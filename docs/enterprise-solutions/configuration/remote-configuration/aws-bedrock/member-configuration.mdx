作为团队成员，你可以把本地开发环境连接到组织的 AWS Bedrock 配置。本指南介绍如何在 VS Code 中配置 AWS credentials，从而通过组织的 Bedrock infrastructure 使用模型。管理员已经完成 provider settings 配置，你只需要添加凭据即可开始。

## 开始之前

要成功连接组织的 AWS Bedrock 配置，需要先准备以下内容。

**已安装并配置 NBG**

需要在 VS Code 中安装 NBG，并登录组织账号。如果尚未安装 NBG，请参考 [安装指南](/getting-started/installing-cline)。

<Info>
**快速检查**：在 VS Code 中打开 NBG panel。如果左下角能看到组织名称，说明已经正确登录。
</Info>

**具备 Bedrock access 的 AWS credentials**

你需要拥有权限的 AWS credentials，才能访问组织配置 region 中的 Bedrock。

<Note>
如果还没有 AWS credentials，请联系 IT 或 cloud team 获取 access keys，或配置具备必要 Bedrock permissions 的 AWS CLI profiles。
</Note>

## 配置步骤

<Steps>
<Step title="打开 NBG Settings">
打开 VS Code，并使用以下任一方式进入 NBG settings panel：

- 点击 NBG panel 中的 settings icon
- 点击聊天区域正下方的 API Provider dropdown，它会显示为 `bedrock.anthropic.claude-sonnet-4-20250514-v1:0` 或类似名称

</Step>

<Step title="选择认证方式">
选择以下任一 credential method 来认证 AWS Bedrock：

<AccordionGroup>
<Accordion title="AWS Bedrock API Key">
使用专门用于 Bedrock access 的 AWS access keys。

[Learn more about AWS Bedrock API Keys](https://docs.aws.amazon.com/bedrock/latest/userguide/api-keys.html)

1. 选择 **API Key** radio button
2. 输入 AWS Access Key ID 和 Secret Access Key
3. 这些凭据会保存在本地，并且只由 VS Code extension 使用
</Accordion>

<Accordion title="AWS Profile">
使用本机已有的 AWS CLI profile。

[Learn more about AWS CLI Profiles](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-profiles.html)

1. 选择 **AWS Profile** radio button
2. 从 `~/.aws/credentials` 文件中选择或输入 profile name
3. NBG 会使用该 profile 关联的 credentials
</Accordion>

<Accordion title="AWS Credentials">
使用默认 AWS credential chain，例如 environment variables、EC2 instance roles 等。

1. 选择 **AWS Credentials** radio button
2. NBG 会使用标准 AWS credential provider chain 从环境中自动检测 credentials
</Accordion>
</AccordionGroup>

<Note>
AWS Region 已由管理员预先配置，不需要在 extension 中设置。
</Note>
</Step>

<Step title="验证配置">
选择 authentication method 后，extension 会为启用的功能显示 checkmarks：

- ✓ Supports images
- ✓ Supports browser use
- ✓ Supports prompt caching

Cross-region inference 和 global inference profile 等额外 settings 会被锁定（显示 lock icon），因为它们由管理员控制。
</Step>

<Step title="测试连接">
在 NBG 中发送一条测试消息，确认凭据可以正确访问已配置的 Bedrock region。

<Tip>
**测试建议**

建议先在 plan mode 中测试连接，确认一切正常后再用于实际任务。
</Tip>
</Step>
</Steps>


## Troubleshooting

**Authentication errors ("Access Denied" or "Invalid Credentials")**

Verify your chosen credential method has the necessary IAM permissions to call Bedrock in the configured region. Required permissions include `bedrock:InvokeModel` and `bedrock:InvokeModelWithResponseStream`. For more information, refer to [AWS Bedrock IAM Permissions](https://docs.aws.amazon.com/bedrock/latest/userguide/security-iam.html).

**Region-related errors or "model not available"**

Ask your administrator to confirm which region is configured for your organization. Ensure your AWS credentials have access to Bedrock in that specific region. [View AWS Global Infrastructure](https://aws.amazon.com/about-aws/global-infrastructure/regions_az/)

**Don't see AWS Bedrock as an option**

Confirm you're signed into the correct NBG organization. Verify your administrator has saved the Bedrock configuration. Try signing out and back into NBG.

**AWS Credentials option not finding credentials**

Verify AWS CLI is installed and configured with `aws configure` ([AWS CLI Installation Guide](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)). Check that credentials are present in `~/.aws/credentials`. For EC2/ECS environments, ensure IAM roles are properly attached. If using environment variables, set `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`.


## 安全最佳实践

配置 AWS credentials 时，遵循以下安全规范：

- Use IAM roles with minimum required permissions ([AWS IAM Best Practices](https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html))
- Rotate access keys regularly if using the API Key method
- Never store credentials in code or version control
- Prefer AWS Profile method for better credential management
- Consider using AWS SSO/federated roles for enhanced security

组织管理员控制可用 models。Extension 会根据 region 中的 Bedrock configuration 自动显示可用 models。可用模型的更多信息见 [AWS Bedrock Model Access documentation](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access.html)。

如需更多帮助，请参考 [AWS Bedrock Documentation](https://docs.aws.amazon.com/bedrock/latest/userguide/what-is-bedrock.html)，并与组织 cloud administrator 协调。
