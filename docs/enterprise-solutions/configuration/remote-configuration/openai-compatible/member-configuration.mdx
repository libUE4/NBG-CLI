作为团队成员，你可以把本地开发环境连接到组织的 OpenAI-compatible endpoint。本指南介绍如何在 VS Code 中配置凭据，从而通过组织配置好的 endpoint 使用模型。管理员已经完成 provider settings 配置，你只需要添加 API key 即可开始。

## 开始之前

要成功连接组织的 OpenAI-compatible endpoint，需要先准备以下内容。

**已安装并配置 NBG**

需要在 VS Code 中安装 NBG，并登录组织账号。如果尚未安装 NBG，请参考 [安装指南](/getting-started/installing-cline)。

<Info>
**快速检查**：在 VS Code 中打开 NBG panel。如果左下角能看到组织名称，说明已经正确登录。
</Info>

**Endpoint 的 API key 或凭据**

你需要 API key 或其他凭据来认证组织配置的 endpoint。对于使用 Azure Identity Authentication 的 Azure Foundry deployments，也可以改用 Azure AD credentials。

<Note>
如果不确定应该使用哪些凭据，请向管理员或 IT 团队确认组织的访问配置。
</Note>

## 配置步骤

<Steps>
<Step title="打开 NBG Settings">
打开 VS Code，并使用以下任一方式进入 NBG settings panel：

- 点击 NBG panel 中的 settings icon
- 点击聊天区域正下方的 API Provider dropdown

</Step>

<Step title="配置凭据">
认证方式取决于管理员如何配置 endpoint：

<AccordionGroup>
<Accordion title="API Key Authentication">
对于大多数 OpenAI-compatible endpoints：

1. 选择或确认已选择 **OpenAI Compatible** provider
2. 在 **API Key** 字段中输入 API key
3. Base URL、custom headers 和其他 settings 已由管理员预先配置
4. 点击 **Save** 保存凭据

<Tip>
API keys 会保存在本地，并且只由 NBG 使用。
</Tip>
</Accordion>

<Accordion title="Azure Identity Authentication (Azure Foundry)">
如果组织使用 Azure AD authentication：

1. 选择或确认已选择 **OpenAI Compatible** provider
2. 确认开发环境中已经登录 Azure
3. Extension 会自动使用 Azure AD credentials
4. 启用 Azure Identity Authentication 时不需要 API key

<Note>
可能需要安装 Azure Account extension 或 Azure CLI 才能解析凭据。
</Note>
</Accordion>
</AccordionGroup>

<Note>
Base URL、custom headers、Azure API version 和 Azure Identity settings 已由管理员预先配置，不需要在 extension 中设置。
</Note>
</Step>

<Step title="验证配置">
配置凭据后，管理员控制的 settings 会被锁定（显示 lock icon），因为它们由组织统一管理。
</Step>

<Step title="测试连接">
在 NBG 中发送一条测试消息，确认凭据可以正确访问已配置的 endpoint。

<Tip>
**测试建议**

开始开发任务前，先用类似 `Hello` 的简单消息验证基础连接。
</Tip>
</Step>
</Steps>

## Troubleshooting

**OpenAI Compatible 未出现在 provider options 中**

确认你登录了正确的 NBG organization。请管理员确认配置已保存，并确保你使用的是最新版本 NBG。

**Authentication errors（`Access Denied` 或 `Invalid API Key`）**

确认 API key 正确且仍然有效。对于使用 Azure Identity Authentication 的 Azure Foundry，请确认开发环境已经登录 Azure，且账号在 Azure OpenAI resource 上拥有合适的 role assignments。

**Connection errors 或 timeouts**

Endpoint URL 由管理员配置。如果遇到连接问题，请向 IT 团队确认网络要求，例如 VPN、firewall rules 等。

**Models 不可用**

可用 models 取决于组织的 endpoint 配置。如果 model dropdown 中缺少预期模型，请联系管理员。

**Configuration changes 没有保存**

确认已经保存凭据。Base URL 和其他管理员控制的 settings 不能在本地修改。

## 安全最佳实践

使用 API credentials 时：

- 妥善保管 API key，不要共享
- 不要把凭据存入代码或版本控制
- 如果怀疑 key 泄露，立即报告管理员
- 遵守组织对该 endpoint 的使用规范

组织管理员控制可用的 endpoint、models 和 settings。Extension 会根据组织 remote configuration 自动应用这些设置。

Azure Foundry 相关信息见 [Azure OpenAI Service documentation](https://learn.microsoft.com/en-us/azure/ai-services/openai/)。其他 endpoints 请参考组织内部文档或联系管理员。
