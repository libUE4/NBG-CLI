## 核心能力

NBG Enterprise 面向生产部署提供五类核心能力，覆盖企业内大规模使用 AI coding agent 时的主要治理需求。

### 默认安全

代码和上下文应留在你的环境中处理。NBG 不要求默认上传代码库、不进行代码库索引，也不使用你的数据训练模型。

### Bring Your Own Inference

复用已有云厂商合同、企业折扣和内部推理平台。NBG 可以直接连接你的 provider 或统一网关，避免把推理采购绑定到单一 SaaS。

可接入的 inference providers 包括：
- AWS Bedrock
- Google Vertex AI
- Azure OpenAI
- Anthropic direct
- OpenAI direct
- Cerebras
- Any OpenAI-compatible endpoint

随着模型发布快速切换。你可以将 Claude Sonnet 作为日常编码模型，将 GPT 系列用于复杂重构，将开源模型用于简单任务。NBG 负责 agent loop，你负责 inference 策略、成本和合规。

### 大规模治理

当大量开发者使用 AI 时，平台团队需要集中控制。散落在个人设备上的 API keys 会带来安全风险和成本失控。

Enterprise governance 提供：
- **SSO authentication**：使用企业身份体系，而不是个人 API keys
- **Role-based access control**：Member/Admin/Owner 角色层级和组织级权限
- **Model and tool controls**：控制各团队可访问的模型和工具
- **Remote configuration**：集中管理开发者配置
- **Usage tracking and observability**：通过 OpenTelemetry 监控 usage、costs 和 performance，并对管理操作做选择性 audit logging

平台团队集中配置，开发者按熟悉的方式工作，同时组织保留治理能力。

### 完整可观测性

将日志和指标导出到现有 observability stack，跨团队跟踪 usage、costs 和 performance。

- **OpenTelemetry export**：对接 Datadog、Grafana、Splunk 等平台
- **Real-time analytics**：跟踪 adoption、performance 和使用模式
- **Cost breakdown**：查看各团队在不同模型上的成本
- **JSON output**：在现有工具中构建自定义 dashboards

按生产系统的可观测标准管理 AI coding agent。

## Deployment

NBG Enterprise 连接你的基础设施，可部署在云环境或企业受控环境中，并按现有安全策略和合规要求配置。

组织部署流程：
1. 配置 NBG Core 连接企业基础设施
2. 设置 SSO、RBAC 和 governance policies
3. 通过现有软件分发方式交付给开发者
4. 通过 observability tools 监控 usage
