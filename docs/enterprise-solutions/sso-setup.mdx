## 概览
NBG Enterprise 可通过 **WorkOS AuthKit** 与你的 identity provider (IdP) 集成，提供 SSO。

本文说明 NBG Enterprise 使用 WorkOS AuthKit 配置 SSO 的高层流程。

如果还没有完成初始 onboarding，请先阅读 [Onboarding](/enterprise-solutions/onboarding)。

## 配置位置
SSO 配置涉及两个位置：

1. **NBG Enterprise Console**
- 用于登录、完成 onboarding，并验证组织 SSO 是否可用。

2. **WorkOS dashboard**
- 用于配置 IdP connection (AuthKit → Connections)。指定管理员会在企业 onboarding 过程中获得访问权限。

## 使用 NBG Enterprise Console
在你的 NBG Enterprise Console 中：
- 完成 sign-in 和 onboarding flows
- 验证用户可以通过 SSO 认证

## 在 WorkOS 中配置 IdP connection

企业 onboarding 期间，指定管理员会收到 WorkOS invitation email，其中包含访问组织 WorkOS dashboard 的链接。

<Frame>
  <img src="/assets/workos-invite-email.png" alt="WorkOS invitation email example" />
</Frame>

要将 IdP 接入 NBG Enterprise，请在 **WorkOS AuthKit** 中配置 identity provider：

1. 在 WorkOS dashboard 中进入 **AuthKit → Connections**
2. 点击 **Add Connection**
3. 选择 identity provider，例如 Okta、Microsoft Entra ID/Azure AD、Google Workspace、Generic SAML/OIDC
4. 按 WorkOS 中对应 provider 的说明完成配置

WorkOS UI 和必填字段会随 provider 不同而变化。详细信息见 WorkOS 文档：
- https://workos.com/docs/authkit/sso

## Keycloak 说明

NBG Enterprise 默认通过 **WorkOS** 集成 SSO。如果使用 **Keycloak** 作为 IdP，建议在 WorkOS 中将 Keycloak 配置为 **Generic SAML** 或 **Generic OIDC** provider，并按 WorkOS 要求填写对应设置。

## 验证
完成 WorkOS 配置后：

1. 从 NBG Enterprise Console 发起一次 SSO sign-in。
2. 确认登录完成，并成功重定向回控制台。

## 排障
- **Redirect URI mismatch**：确认 WorkOS 中配置的 redirect/callback URL 与 NBG Enterprise onboarding 期间提供的 URL 一致。

更多排障信息请参考 WorkOS 文档：
- https://workos.com/docs/authkit/sso
