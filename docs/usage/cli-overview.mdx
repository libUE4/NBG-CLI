## 前置条件

- 已安装 NBG CLI：`npm i -g nbg`
- 已配置 provider 认证：`nbg auth`（见认证配置指南）

## 快速开始

```bash
# 交互式会话
nbg

# 立即运行一个任务
nbg "把这个模块重构为 async/await"

# 为脚本输出结构化 JSON
nbg --json "列出 TODO 注释"
```

## Headless mode

脚本、自动化和可处理输出场景使用 headless mode。

使用 `--json`、stdin pipe 或 stdout redirect 时会触发 headless mode。

#### 何时触发 headless

| 调用方式 | 原因 |
|---|---|
| `nbg --json "task"` | JSON output mode |
| `cat file \| nbg "task"` | stdin 通过 pipe 传入 |
| `nbg "task" > output.txt` | stdout 被重定向 |

```bash
# CI/script 风格执行
git diff | nbg "审查这些变更"

# 便于解析的 JSON
nbg --json "总结这个 changelog" | jq -r '.text'
```

参考：[Headless mode](#headless-mode)

### 自主执行

需要完全无人值守运行时，使用自动批准：

```bash
nbg --auto-approve true "运行测试并修复失败"
```

Headless runs 同样支持模式选择：

```bash
# 先规划
nbg -p "设计迁移计划"

# 立即执行（默认）
nbg "应用迁移"
```

<Warning>
自主执行可能在没有进一步确认的情况下修改文件并运行命令。请使用干净分支并审查结果。
</Warning>

## 常用命令

```bash
nbg --help
nbg <command> --help
```

| 命令 | 用途 |
|---|---|
| `nbg` | 启动交互模式或运行 prompt |
| `nbg auth` | 认证并设置 provider/model |
| `nbg config` | 打开交互式配置视图 |
| `nbg mcp` | 管理 MCP servers |
| `nbg doctor` | 诊断并修复配置问题 |
| `nbg history` | 查看和管理任务历史 |
| `nbg schedule` | 管理定时任务 |
| `nbg hub` | 管理本地 hub 守护进程 |
| `nbg kanban` | 启动 Kanban 看板应用 |

## 常用全局参数

以 [CLI 参考](/cli/cli-reference) 为准。

| 参数 | 用途 |
|---|---|
| `-p, --plan` | 以 Plan mode 启动 |
| `--auto-approve <boolean>` | 全局 tool 自动批准（`true`/`false`，默认 `true`） |
| `-m, --model <model>` | 覆盖本次运行的模型 |
| `-P, --provider <id>` | 覆盖本次运行的 provider |
| `-c, --cwd <path>` | 设置工作目录 |
| `--config <dir>` | 使用指定配置目录 |
| `--data-dir <dir>` | 使用隔离本地状态 |
| `--json` | 输出 newline-delimited JSON messages |
| `--thinking <level>` | 设置推理强度：`none\|low\|medium\|high\|xhigh`（默认 `medium`） |
| `-t, --timeout <seconds>` | 设置任务超时 |

## 自动化模式

### 通过 pipe 输入上下文

```bash
cat README.md | nbg "总结关键设置步骤"
git diff | nbg "审查潜在回归"
```

### 串联任务

```bash
git diff | nbg "解释这些变更" | nbg "写一条提交信息"
```

### 限制命令执行

```bash
export CLINE_COMMAND_PERMISSIONS='{"allow": ["npm *", "git *"], "deny": ["rm -rf *", "sudo *"]}'
```

### 在任务中引用图片

```bash
nbg -i "修复 @./screenshot.png 中显示的布局问题"

# 或直接 inline 引用
nbg "修复 @./design-mockup.png 中显示的 UI"
```

### 设置执行超时

```bash
nbg --timeout 600 "运行完整测试套件"
```

## JSON 输出格式

使用 `--json` 时，每一行都是一个 JSON message object。

```json
{"type":"say","text":"我现在创建文件。","ts":1760501486669,"say":"text"}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `type` | `"ask"` 或 `"say"` | 消息类别 |
| `text` | `string` | 消息内容 |
| `ts` | `number` | Unix 毫秒时间戳 |
| `say` | `string` | `type` 为 `"say"` 时的子类型 |
| `ask` | `string` | `type` 为 `"ask"` 时的子类型 |
| `reasoning` | `string` | 可选模型推理内容 |
| `partial` | `boolean` | 流式输出标记 |
