自动批准让你决定哪些 NBG 操作可以跳过逐次确认。它适合处理日常低风险操作，同时保留对高风险动作的控制。

如果你经常为同一类安全操作反复点击批准，可以启用自动批准减少打断。

## 工作方式

自动批准会按 tool call 逐次判断。NBG 准备读取文件、编辑文件、运行命令或使用浏览器时，会检查该类别对应的自动批准设置。

需要注意的细节：

- **Workspace 内外**："Read all files" 和 "Edit all files" 只是在基础开关上扩展范围。如果基础开关关闭，"all files" 选项不会生效。
- **终端命令**：NBG 会把命令区分为安全命令和需要批准的命令。"Execute safe commands" 覆盖安全命令，"Execute all commands" 会扩展到标记为需要批准的命令。
- **通知**：启用后，当需要批准，或自动批准的终端命令运行超过 30 秒时，NBG 会发送系统通知。

## 权限项

| 设置 | 允许的操作 |
|---------|----------------|
| Read project files | 读取、列出和搜索 workspace 内文件 |
| Read all files | 读取 workspace 外文件（需要基础开关开启） |
| Edit project files | 在 workspace 内创建和编辑文件 |
| Edit all files | 编辑 workspace 外文件（需要基础开关开启） |
| Execute safe commands | 运行标记为安全的终端命令 |
| Execute all commands | 运行需要批准的命令（需要基础开关开启） |
| Use the browser | 使用浏览器工具抓取和搜索网页 |
| Use MCP servers | 使用 MCP tools 和 resources |
| Enable notifications | 为长时间运行命令发送通知 |

<Warning>
"Read all files" 和 "Edit all files" 只有在对应基础开关开启时才有效。它们会把访问范围扩展到 workspace 外。
</Warning>

## 安全命令与需批准命令

NBG 不使用固定 allowlist。模型会根据命令和参数为每条命令标记 `requires_approval`。下面只是常见示例，不是绝对保证。

**通常会被视为安全：**
- `npm run build`、`npm test`：构建或测试输出
- `git status`、`ls -la`、`cat package.json`：只读命令

**通常需要批准：**
- `npm install <pkg>`：修改依赖
- `rm -rf <path>`：删除文件
- `mv <a> <b>`：移动文件，可能覆盖目标
- `sed -i ...`：原地编辑文件

## 推荐设置

建议默认配置：
- 启用 **Read project files**
- 在没有明确理由前，先关闭编辑、命令、浏览器和 MCP 自动批准

如果启用编辑自动批准，建议同时使用 [Checkpoints](/core-workflows/checkpoints)，方便快速回滚。

## YOLO 模式

YOLO 模式会自动批准所有操作：文件修改、终端命令、浏览器动作、MCP tools 和模式切换。

<Note>
**警告：该模式风险很高。** YOLO 模式会关闭所有批准检查。NBG 会直接执行它决定执行的操作，不再逐次请求确认。
</Note>

### 自动批准范围

启用 YOLO 模式后：
- 系统任意位置的所有文件操作
- 所有终端命令，包括潜在破坏性命令
- 浏览器操作
- MCP server tools
- 模式切换（Plan 到 Act）

### 适用场景

**快速原型**：需要零阻力执行，并且可以接受潜在错误，适合一次性实验。

**可信重复任务**：你已经验证过 NBG 的执行方式，希望去掉批准开销。

**演示场景**：需要展示 NBG 能力，并避免中途被批准弹窗打断。

### 可能的风险

NBG 可能会：
- 在没有警告的情况下删除重要文件
- 执行修改系统设置的命令
- 向外部服务发起网络请求
- 覆盖配置文件
- 安装或卸载依赖包
- 提交并推送版本控制变更

### YOLO 模式最佳实践

- **先使用隔离环境。** 优先在一次性项目或 sandbox 环境中验证。
- **给出具体请求。** 模糊指令加无限权限会带来不可预测结果。
- **持续观察输出。** NBG 仍会展示正在做什么，注意终端输出和文件变化。
- **保持版本控制可用。** Git 是最后的回滚保障。

### 启用 YOLO 模式

进入 NBG Settings，然后打开 Features，勾选 "YOLO Mode"。启用后不会再出现确认对话框，NBG 会立即自动批准所有动作。取消勾选即可关闭。
