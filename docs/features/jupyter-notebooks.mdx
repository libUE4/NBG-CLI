NBG 支持 Jupyter notebooks（`.ipynb` 文件）的 AI 辅助编辑，并能理解 cell 级上下文。该能力适合数据科学、实验分析和 notebook 型工作流。

## 开始使用

Jupyter notebook 支持来自继承运行时的 IDE 能力。使用前需要在 VS Code 中启用 Jupyter notebook extension。打开 `.ipynb` 文件后，可以在 notebook 界面中看到 AI 辅助按钮。

## 使用方式

### 生成 Cell

点击 notebook toolbar 中的 sparkle 图标，让 AI 生成新的 cell。

<Frame>
  <img src="/assets/jupyter-generate-cell.gif" alt="Generating a new Jupyter cell with AI assistance" />
</Frame>

AI 会接收周围 cells 的上下文，因此能理解当前作用域中的变量和 imports。你可以直接引用已有 DataFrames、functions 和其他对象，不需要重新解释。

**示例 prompt：** "创建一个 heatmap，可视化数值列的相关矩阵"

生成的 cell 会按 notebook JSON 结构插入，并保留 metadata，插入后即可执行。

### 解释 Cell

点击任意 cell title bar 中的 Explain button，可以获取该 cell 的详细解释。

适合以下场景：

- 回看旧 notebooks
- 理解队友的分析过程
- 分析复杂数据转换

NBG 会提取完整 cell 上下文，包括 outputs。因此解释可以引用列名、行数和计算结果等真实输出。

### 改进 Cell

点击任意 cell title bar 中的 Improve button，可以让 AI 提出改进建议。

<Frame>
  <img src="/assets/jupyter-explain-improve-cell.gif" alt="Explaining and improving a Jupyter cell with AI" />
</Frame>

适合用于：

- 优化较慢的 pandas 操作
- 添加错误处理
- 重构以提升可读性
- 将循环转换为向量化操作

NBG 会在保留 cell 位置和 metadata 的前提下提出修改，并解释变更内容和原因。

## Cell 上下文如何工作

Jupyter notebooks 与普通文件不同，本质是包含 cell 数组的 JSON 文档。每个 cell 都有自己的 type、source content、metadata、execution count 和 outputs。

使用 Jupyter 命令时，NBG 会提取结构化上下文，包括：

- **Cell type**（code、markdown 或 raw）
- **Source content**，以 lines 数组表示
- **Cell metadata** 和配置
- code cells 的 **execution count**
- **Outputs**，包括 data、text 和 error traces

这种结构化表示能让 AI 不只理解代码本身，也理解它在 notebook 中的位置和实际输出。

### 保留 JSON 结构

NBG 会谨慎处理 cell JSON 结构，目标是：

- 保持 cell 边界完整
- 保留 execution counts
- 维护 cell metadata
- 保持 outputs 与原 cell 的关联

系统提示会要求 AI 保留 notebook 结构，但你仍应审查变更，确认 notebook 格式正确。

## 快捷键

可以把这些命令绑定到快捷键，提高使用效率：

1. 打开 VS Code keyboard shortcuts（Cmd/Ctrl + K, Cmd/Ctrl + S）
2. 搜索 `cline.jupyterGenerateCell`、`cline.jupyterExplainCell` 或 `cline.jupyterImproveCell`
3. 分配你偏好的快捷键组合

这些 command id 当前仍沿用继承运行时的 `cline.*` 兼容命名。

## 最佳实践

**生成 Cell：**
- 明确说明你希望 cell 完成什么
- 直接按名称引用已有变量（AI 可以看到它们）
- 如有偏好，说明希望使用的库，例如 "use seaborn" 或 "use plotly"

**解释 Cell：**
- 对已经执行过的 cells 效果最好，因为 outputs 会提供额外上下文
- 适合理解 pandas groupby/merge 这类复杂链式操作

**改进 Cell：**
- 指出希望改进的方面，例如性能、可读性或错误处理
- AI 会解释建议变更

## 限制

- Notebook 支持需要在 VS Code 中启用 Jupyter notebook extension
- Cell 上下文提取依赖 VS Code notebook API
- 超大 notebooks 可能需要超过部分模型可高效处理的上下文
