Subagents 让 NBG 可以启动多个聚焦研究 agent 并行工作。每个 subagent 都有自己的 prompt 和上下文窗口，可以独立探索代码库，并把详细报告返回给主 agent。这样可以快速收集广泛信息，同时保持主 agent 上下文更干净。

<Tip>
  Subagents 仍是实验性能力，未来版本的行为可能调整。
</Tip>

## 工作方式

当 NBG 使用 `use_subagents` tool 时，会同时启动多个独立 agent。每个 subagent：

- 拥有自己的研究 prompt
- 使用独立上下文窗口和 token 预算
- 可以读取文件、搜索代码、列目录、运行只读命令并使用 skills
- 不能编辑文件、使用浏览器、访问 MCP servers 或继续创建嵌套 subagents
- 返回结果时会突出主 agent 下一步最该读取的相关文件路径

Subagent 的 token 和 API 成本会按 subagent 单独跟踪，并计入任务总成本。运行时可以在 chat UI 中查看每个 subagent 的 tool calls、tokens 和成本统计。

## 启用 Subagents

Subagents 默认启用。NBG 会判断并行研究是否值得额外开销，你不需要在 prompt 中显式开启。需要关闭时，在 Settings -> Features -> Agent 中禁用 `use_subagents` tool。

该设置适用于所有入口，包括 VS Code、JetBrains 和 CLI。

## 使用方式

启用 subagents 后，NBG 会在任务适合并行探索时自动使用它们。你也可以在 prompt 中明确要求并行研究。

示例 prompt：

- "使用 subagents 探索认证流程如何工作，以及数据库模型定义在哪里"
- "启动 subagents 分别调查 API routes、测试设置和部署配置"
- "我刚接触这个代码库。使用 subagents 梳理主入口、路由层和数据访问模式"

每个 subagent prompt 都应该描述一个聚焦的研究问题。NBG 会并行运行它们并综合结果。任务较小时，也可以只运行一个 subagent，避免并行发现带来不必要开销。

## 自动批准行为

Subagents 遵循 **Read project files** 自动批准权限。如果你在 [自动批准](/features/auto-approve) 中启用了 "Read project files"，subagent 启动也会被自动批准。

如果关闭自动批准，NBG 会在启动 subagents 前请求确认，并展示准备发送的 prompts。

## Subagents 能做什么

Subagents 是只读研究 agent。它们可以访问：

| Tool | 用途 |
|------|---------|
| `read_file` | 读取文件内容 |
| `list_files` | 列出目录内容 |
| `search_files` | 使用正则搜索文件 |
| `list_code_definition_names` | 列出顶层 classes、functions 和 methods |
| `execute_command` | 运行只读命令（`ls`、`grep`、`git log`、`git diff` 等） |
| `use_skill` | 加载并激活 skills |

Subagents 不能写文件、应用 patch、使用浏览器、访问 MCP servers 或执行 web search。它们也不能继续创建自己的 subagents。

<Note>
  Subagents 运行的命令会在后台执行，并限制为只读操作。Subagents 不会运行会修改文件或系统状态的命令。
  Subagents 也适合先使用命令管道和过滤器缩小输出，再读取文件，例如 `rg ... | sort | uniq`。
</Note>

## 适用场景

当你需要同时从代码库多个区域收集广泛上下文时，subagents 最有价值：

- **熟悉陌生项目**：让 subagents 并行梳理架构、关键入口和数据流。
- **调查横切关注点**：让不同 subagents 同时追踪认证、日志和错误处理。
- **修改前研究**：在改代码前，让 subagents 从相关文件收集上下文，帮助主 agent 在不消耗大量上下文的情况下做出更稳的修改。
- **大型代码库**：顺序读取大量文件会占用主 agent 上下文时，subagents 可以更广泛地探索。

如果任务很小且你已经知道要看哪些文件，subagents 会增加不必要开销。直接让 NBG 处理即可。
