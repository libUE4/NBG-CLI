MCP (Model Context Protocol) 让 NBG 可以通过 MCP servers 使用外部工具和数据源。

## MCP 提供什么

- 将 NBG 连接到外部 APIs 和 services
- 添加内置 NBG tools 之外的自定义工具
- 使用本地 servers 或 remote hosted servers

## 快速开始

1. 在 NBG 中打开 **MCP Servers**
2. 添加 server（从 Marketplace 或手动添加）
3. 配置 credentials/environment variables
4. 确认 tools 出现，并测试一次 tool call

## 添加 servers

### 方式 1：MCP 市场

如果可用，可以使用 NBG MCP 市场一键安装。
1. 在 NBG panel 中点击 MCP Servers icon（top toolbar 中的 stacked server icon）。
2. 打开 Marketplace tab。

### 方式 2：手动配置

编辑 MCP config file 并添加配置：

- **CLI:** `~/.cline/mcp.json`
- **IDE extensions:**
  1. 在 NBG panel 中点击 **MCP Servers** icon。
  2. 打开 **Configure** tab。
  3. 点击 **Configure MCP Servers**。
  4. 打开 extension 使用的 MCP settings JSON 后，在 `mcpServers` 下添加或更新 entries。
  - 如果添加 hosted endpoint，而不是直接编辑 JSON，使用 **Remote Servers** tab：
    1. 输入 **Server Name**（任意唯一 label）。
    2. 输入 **Server URL**（完整 endpoint URL）。
    3. 选择 **Transport Type**：
       - **Streamable HTTP** (recommended)
       - **SSE (Legacy)**
    4. 点击 **Add Server**。

  - Server config shape：
    - **Local (STDIO)** server 使用 `command` + `args`
    - **Remote (HTTP/SSE)** server 使用 `url`

### CLI MCP 向导

在 CLI 中运行：

```bash
nbg mcp
```

向导支持：

| 操作 | 说明 |
|---|---|
| List servers | 查看已配置 servers 及启用状态 |
| Add server | 创建新的 MCP server entry |
| Edit server | 修改已有 server |
| Enable/Disable | 启用或禁用 server，不删除配置 |
| Delete server | 永久删除 server |

添加 server 时，CLI 会提示输入 server name、transport type、command/args（stdio）或 URL/headers（remote transports）。

## Configuration examples

### Local server (STDIO)

```json
{
  "mcpServers": {
    "local-server": {
      "command": "node",
      "args": ["/path/to/server.js"],
      "env": {
        "API_KEY": "your_api_key"
      },
      "disabled": false,
      "autoApprove": []
    }
  }
}
```

### Remote server (HTTP/SSE)

```json
{
  "mcpServers": {
    "remote-server": {
      "url": "https://example.com/mcp",
      "headers": {
        "Authorization": "Bearer your-token"
      },
      "disabled": false,
      "autoApprove": []
    }
  }
}
```

## Transport types

- **STDIO**：本地进程，延迟低，本地设置更简单
- **Remote HTTP/SSE**：hosted endpoint，集中部署，支持 multi-client usage

本地 tools 优先用 STDIO，共享 hosted services 使用 remote transport。

## 管理 servers

在 MCP settings 中可以：

- 启用或禁用 servers
- 重启无响应的 servers
- 设置 request timeouts
- 移除 servers

## 安全基础

- 只安装可信 servers
- 将 secrets 存入环境变量
- 只为安全 tools 配置 `autoApprove`
- 批准前审查 tool calls

## 故障排查

| 问题 | 处理方式 |
|---|---|
| Server 无法连接 | 检查 command/URL、server process 状态和端口 |
| Tools 缺失 | 确认 server 成功启动，并暴露了 tools |
| Auth errors | 重新检查 API keys/tokens 和 required headers |
| Timeout errors | 增加 MCP timeout，并直接测试 server response |

## CLI

MCP 也可在 NBG CLI 中使用。通过 CLI MCP settings 配置 servers，并复用相同 server definitions。

也可以用非交互方式列出 servers：

```bash
nbg config mcp
nbg config mcp --json
```
