每个 NBG API 请求都需要在 `Authorization` header 中携带 Bearer token。

## 认证方式

常见认证方式如下：

| 方式 | 使用场景 | 获取方式 |
|--------|----------|---------------|
| **API key** | 直接 API calls、scripts、CI/CD | 由你的 NBG API gateway 或企业控制台签发 |
| **Account auth token** | IDE、CLI 或兼容账号体系 | 登录后自动生成和管理 |

两种方式使用相同 header 格式：

```bash
Authorization: Bearer YOUR_TOKEN
```

## API Keys

API keys 是程序化访问的推荐方式。

### 创建 Key

<Steps>
  <Step title="打开控制台">
    打开你的 NBG API gateway 或企业控制台。
  </Step>
  <Step title="进入 API Keys">
    进入 **Settings** > **API Keys**。
  </Step>
  <Step title="创建并复制">
    创建新 key，并立即复制保存。多数控制台不会再次显示完整 key。
  </Step>
</Steps>

### 删除 Key

可以随时在 **Settings** > **API Keys** 页面撤销 API key。删除后的 key 会立即失效。

如果部署提供 Enterprise API，也可以通过 API 管理 keys：

```bash
export NBG_API_BASE_URL="https://<your-nbg-api>/api/v1"

# 列出 keys
curl "$NBG_API_BASE_URL/api-keys" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 删除 key
curl -X DELETE "$NBG_API_BASE_URL/api-keys/KEY_ID" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Account Auth Tokens

当你在 NBG IDE 入口或 CLI 中登录时，account auth token 会自动生成并管理。通常不需要手动处理这些 tokens。

NBG CLI 会在以下认证流程中使用这些 tokens：

```bash
# 交互式登录
nbg auth

# 或使用 API key 快速配置兼容 provider
nbg auth -p openai-compatible -k "YOUR_API_KEY" -m anthropic/claude-sonnet-4-6
```

完整认证参数见 [CLI 参考](/cli/cli-reference)。

## 安全最佳实践

**建议：**
- 将 API keys 存入环境变量或 secrets manager
- 开发和生产使用不同 keys
- 定期轮换 keys
- 删除不再使用的 keys

**不要：**
- 将 keys 提交到版本控制
- 在聊天或邮件中分享 keys
- 将 keys 嵌入浏览器或移动端代码
- 在应用输出中记录 keys

### 使用环境变量

```bash
# 设置 key
export NBG_API_KEY="your_api_key_here"
export NBG_API_BASE_URL="https://<your-nbg-api>/api/v1"

# 在请求中使用
curl -X POST "$NBG_API_BASE_URL/chat/completions" \
  -H "Authorization: Bearer $NBG_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "anthropic/claude-sonnet-4-6", "messages": [{"role": "user", "content": "Hello"}]}'
```

### 使用 .env 文件

```bash
# .env (add to .gitignore)
NBG_API_KEY=your_api_key_here
NBG_API_BASE_URL=https://<your-nbg-api>/api/v1
```

```python
import os
from openai import OpenAI

client = OpenAI(
    base_url=os.environ["NBG_API_BASE_URL"],
    api_key=os.environ["NBG_API_KEY"],
)
```

## 自定义 Headers

NBG API 可以接受用于追踪和识别的可选 headers：

| Header | 说明 |
|--------|-------------|
| `HTTP-Referer` | 你的应用 URL，帮助追踪用量来源。 |
| `X-Title` | 你的应用名称，可显示在使用日志中。 |
| `X-Task-ID` | 唯一任务标识，便于排查和关联请求。 |
