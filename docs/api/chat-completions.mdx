Chat Completions endpoint 根据对话消息生成模型响应。NBG API 使用 [OpenAI Chat Completions](https://platform.openai.com/docs/api-reference/chat/create) 兼容格式，便于复用现有 SDK、代理和网关。

## Endpoint

```
POST https://<your-nbg-api>/api/v1/chat/completions
```

## Request Headers

| Header | Required | 说明 |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer YOUR_API_KEY` |
| `Content-Type` | Yes | `application/json` |
| `HTTP-Referer` | No | 应用 URL，用于用量跟踪 |
| `X-Title` | No | 应用名称，用于日志和用量展示 |

## Request Body

| Parameter | Type | Required | Default | 说明 |
|-----------|------|----------|---------|-------------|
| `model` | string | Yes | | `provider/model` 格式的 model ID，见 [模型](/api/models)。 |
| `messages` | array | Yes | | 对话消息。每条消息包含 `role` (`system`, `user`, `assistant`) 和 `content`。 |
| `stream` | boolean | No | `true` | 以 Server-Sent Events stream 返回响应。 |
| `tools` | array | No | | OpenAI 格式的 tool/function definitions。 |
| `temperature` | number | No | Model default | 采样温度，范围 0.0 到 2.0。值越低越确定。 |

### Message Format

`messages` 数组中的每条消息使用以下结构：

```json
{
  "role": "user",
  "content": "你的消息"
}
```

**Roles：**

| Role | 用途 |
|------|---------|
| `system` | 设置模型行为和角色，通常放在数组第一条。 |
| `user` | 用户输入。 |
| `assistant` | 之前的模型响应，用于多轮对话。 |

### 多轮对话

包含历史消息即可保留上下文：

```json
{
  "model": "anthropic/claude-sonnet-4-6",
  "messages": [
    {"role": "system", "content": "You are a helpful coding assistant. Answer in Chinese."},
    {"role": "user", "content": "JavaScript 中的闭包是什么？"},
    {"role": "assistant", "content": "A closure is a function that..."},
    {"role": "user", "content": "可以给一个例子吗？"}
  ]
}
```

## Streaming Response

当 `stream: true`（默认值）时，响应是一组 [Server-Sent Events](https://developer.mozilla.org/en-US/docs/Web/API/Server-Sent_Events)：

```
data: {"id":"gen-abc123","choices":[{"delta":{"role":"assistant"},"index":0}],"model":"anthropic/claude-sonnet-4-6"}

data: {"id":"gen-abc123","choices":[{"delta":{"content":"The capital"},"index":0}],"model":"anthropic/claude-sonnet-4-6"}

data: {"id":"gen-abc123","choices":[{"delta":{"content":" of France"},"index":0}],"model":"anthropic/claude-sonnet-4-6"}

data: {"id":"gen-abc123","choices":[{"delta":{"content":" is Paris."},"index":0,"finish_reason":"stop"}],"model":"anthropic/claude-sonnet-4-6","usage":{"prompt_tokens":14,"completion_tokens":8,"cost":0.000066}}

data: [DONE]
```

每个 `data:` 行包含一个 JSON chunk。关键字段如下：

| Field | 说明 |
|-------|-------------|
| `id` | 生成 ID，所有 chunks 保持一致 |
| `choices[0].delta.content` | 当前 chunk 的新增文本 |
| `choices[0].delta.reasoning` | reasoning/thinking 内容，适用于 reasoning models |
| `choices[0].finish_reason` | 完成时为 `stop`，失败时为 `error` |
| `usage` | token 用量和成本，通常在最后一个 chunk 中返回 |

### Usage Object

最后一个 chunk 会包含 token 用量和成本：

```json
{
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 42,
    "prompt_tokens_details": {
      "cached_tokens": 0
    },
    "cost": 0.000315
  }
}
```

| Field | 说明 |
|-------|-------------|
| `prompt_tokens` | 输入 token 总数 |
| `completion_tokens` | 输出 token 总数 |
| `prompt_tokens_details.cached_tokens` | 命中缓存的 tokens，可降低成本 |
| `cost` | 当前请求的美元成本 |

## Non-Streaming Response

当 `stream: false` 时，响应是单个 JSON object：

```json
{
  "id": "gen-abc123",
  "model": "anthropic/claude-sonnet-4-6",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "The capital of France is Paris."
      },
      "finish_reason": "stop",
      "index": 0
    }
  ],
  "usage": {
    "prompt_tokens": 14,
    "completion_tokens": 8
  }
}
```

## Tool Calling

可以使用 OpenAI function calling 格式定义模型可调用的 tools：

```json
{
  "model": "anthropic/claude-sonnet-4-6",
  "messages": [
    {"role": "user", "content": "What's the weather in San Francisco?"}
  ],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "get_weather",
        "description": "Get the current weather for a location",
        "parameters": {
          "type": "object",
          "properties": {
            "location": {
              "type": "string",
              "description": "City and state, e.g. San Francisco, CA"
            }
          },
          "required": ["location"]
        }
      }
    }
  ]
}
```

当模型决定调用 tool 时，响应会包含 `tool_calls` 数组：

```json
{
  "choices": [
    {
      "message": {
        "role": "assistant",
        "tool_calls": [
          {
            "id": "call_abc123",
            "type": "function",
            "function": {
              "name": "get_weather",
              "arguments": "{\"location\": \"San Francisco, CA\"}"
            }
          }
        ]
      },
      "finish_reason": "tool_calls"
    }
  ]
}
```

tool 调用后继续对话时，需要把 tool result 一并传回：

```json
{
  "messages": [
    {"role": "user", "content": "What's the weather in San Francisco?"},
    {"role": "assistant", "tool_calls": [{"id": "call_abc123", "type": "function", "function": {"name": "get_weather", "arguments": "{\"location\": \"San Francisco, CA\"}"}}]},
    {"role": "tool", "tool_call_id": "call_abc123", "content": "{\"temperature\": 62, \"condition\": \"foggy\"}"},
  ]
}
```

## Reasoning Models

部分模型支持 extended thinking 或 reasoning。使用这些模型时，streaming delta 中可能包含 reasoning 内容：

```json
{"choices":[{"delta":{"reasoning":"Let me think about this step by step..."}}]}
```

Reasoning tokens 与主输出分开，通常出现在 `delta.reasoning` 字段中。部分 provider 会通过 `delta.reasoning_details` 返回加密 reasoning blocks，可在后续请求中传回以保留 reasoning trace。

<Note>
  并非所有模型都支持 reasoning。支持情况见 [模型](/api/models)。
</Note>

## 完整示例

```bash
export NBG_API_BASE_URL="https://<your-nbg-api>/api/v1"
export NBG_API_KEY="YOUR_API_KEY"

curl -X POST "$NBG_API_BASE_URL/chat/completions" \
  -H "Authorization: Bearer $NBG_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "anthropic/claude-sonnet-4-6",
    "messages": [
      {"role": "system", "content": "You are a concise assistant. Answer in one sentence."},
      {"role": "user", "content": "Explain what an API is."}
    ],
    "stream": true
  }'
```
