---
title: "Cline 兼容 provider"
description: "在 NBG 中使用继承运行时保留的 Cline OAuth provider。"
---

NBG 当前保留 provider id `cline` 作为兼容层。它适合复用已有 OAuth 配置、测试继承路径，或在迁移期间继续使用旧账号体系。

商业化部署优先使用 OpenAI-compatible 或 BYOK provider；只有明确需要兼容旧账号时才选择该 provider。

## 何时使用

- 需要复用已有 OAuth 登录
- 需要验证从 Cline 底座继承的 provider 行为
- 正在从旧配置迁移到 NBG

## Setup

<Steps>
  <Step title="打开 NBG 设置">
    点击 NBG 面板中的设置图标。
  </Step>

  <Step title="选择 provider">
    Set **API Provider** to **Cline**.
  </Step>

  <Step title="登录">
    点击 **Sign In** 并在浏览器中完成认证。
  </Step>

  <Step title="选择模型">
    从 **Model** 下拉框选择模型。
  </Step>

  <Step title="验证">
    发送一条测试消息。如果 NBG 正常响应，配置完成。
  </Step>
</Steps>

## CLI

```bash
nbg auth cline
```

兼容 provider 的计费、组织和用量信息由对应旧账号体系管理。NBG 仓库不把旧 Dashboard 作为默认商业化入口。
