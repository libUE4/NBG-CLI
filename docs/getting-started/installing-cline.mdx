## 选择安装路径

- [CLI](#cli) — 终端交互和自动化工作流
- [IDE Extension](#ide-extension) — VS Code、Cursor、JetBrains、Windsurf、VSCodium、Antigravity
- [Kanban](#kanban) (preview) — 用看板管理多个 agent 任务
- [SDK](#sdk) — 基于当前 `@cline/sdk` 兼容层构建

## IDE Extension

需要编辑器 UI 时使用这一方式。NBG IDE 扩展仍在从 Cline 兼容层迁移，正式发布前优先使用 CLI；需要 IDE 集成时可基于本仓库构建，或使用兼容扩展并完成 provider 配置。

<Tabs>
  <Tab title="VS Code / Cursor / Windsurf / VSCodium / Antigravity">
    <Steps>
      <Step title="打开扩展面板">
        按 `Ctrl/Cmd + Shift + X`。
      </Step>
      <Step title="搜索扩展">
        搜索 `NBG`；如果尚未发布，使用本仓库构建的 VSIX。
      </Step>
      <Step title="安装">
        点击 **Install**，或通过编辑器的 VSIX 安装入口安装本地构建包。
      </Step>
      <Step title="打开 NBG">
        使用 NBG activity bar 图标，或从 Command Palette 打开 NBG 面板。
      </Step>
      <Step title="配置 provider">
        安装后在 NBG 设置中完成 provider 配置。

        [配置认证](/getting-started/authorizing-with-cline)
      </Step>
    </Steps>

    <Note>
    Windsurf 和 VSCodium 使用 Open VSX，安装流程相同。
    </Note>
  </Tab>

  <Tab title="JetBrains">
    <Steps>
      <Step title="打开 Plugins Marketplace">
        进入 **Settings** → **Plugins** → **Marketplace**。
      </Step>
      <Step title="安装插件">
        搜索 `NBG`；如果尚未发布，使用本仓库构建的插件包。
      </Step>
      <Step title="打开 NBG">
        通过 **View** → **Tool Windows** 打开 NBG。
      </Step>
      <Step title="配置 provider">
        安装后在 NBG 设置中完成 provider 配置。

        [配置认证](/getting-started/authorizing-with-cline)
      </Step>
    </Steps>

    发布前不要把旧 Marketplace 页面作为 NBG 安装入口。
  </Tab>
</Tabs>

## CLI

需要终端交互和自动化任务时使用 CLI。

<Steps>
  <Step title="安装 Node.js">
    安装 Node.js 20+，推荐 Node.js 22。
  </Step>
  <Step title="安装 CLI">
    ```bash
    npm install -g @nbg/cli
    ```
  </Step>
  <Step title="配置认证">
    ```bash
    nbg auth
    ```
  </Step>
  <Step title="运行 NBG">
    ```bash
    nbg
    # 或
    nbg "你的任务"
    ```
  </Step>
</Steps>

更多信息见 [CLI 参考](/cli/cli-reference)。

## Kanban

需要用任务看板管理 agent 执行时使用 Kanban。

<Steps>
  <Step title="安装 CLI">
    先安装 NBG CLI。
  </Step>
  <Step title="启动 Kanban">
    ```bash
    nbg kanban
    ```
  </Step>
</Steps>

更多信息见 [Kanban](/usage/kanban)。

## SDK

需要在 NBG 底座上构建自己的 app 或 agent 时使用 SDK。包名当前保留 `@cline/sdk` 作为兼容层。

<Steps>
  <Step title="创建项目">
    ```bash
    mkdir my-agent && cd my-agent
    npm init -y
    ```
  </Step>
  <Step title="安装 SDK">
    ```bash
    npm install @cline/sdk
    ```
  </Step>
  <Step title="构建并运行">
    浏览 SDK 示例，运行你的第一个 agent。
  </Step>
</Steps>

从这里开始：[SDK 示例](/sdk/examples)。
