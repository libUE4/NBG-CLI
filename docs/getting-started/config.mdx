NBG 当前沿用 Cline 运行时的配置目录作为兼容层，配置分为两个作用域：

- **全局配置** 位于 `~/.cline/`（当前全局兼容目录，适用于 NBG CLI、IDE 和 SDK）
- **项目配置** 位于 `.cline/`（只作用于当前 workspace）

## Configuration Directory Layout

NBG 目前把共享配置存储在几个固定位置。主目录仍是 `~/.cline/`，结构化状态位于 `~/.cline/data/`：

```text
~/.cline/
  data/
    settings/
      providers.json           # API keys and provider configuration
      global-settings.json     # 全局设置
      cline_mcp_settings.json  # MCP settings
    teams/                     # 团队状态
    sessions/                  # 会话数据
    db/                        # SQLite databases (for example cron.db)
    workflows/                 # 全局 workflows
  rules/                       # 全局规则
  hooks/                       # 全局 hooks
  skills/                      # 全局技能
  agents/                      # 全局 agent 定义
  plugins/                     # 全局插件 (.js, .ts)
  cron/                        # 全局 cron specs
```

代码还支持这些额外全局搜索路径，用于兼容旧安装：

```text
~/Documents/Cline/
  Rules/                       # Additional global rules
  Hooks/                       # Additional global hooks
  Plugins/                     # Additional global plugins
  Workflows/                   # Additional global workflows
```

项目级配置位于仓库根目录的 `.cline/`：

```text
.cline/
  rules/                       # 项目规则
  skills/                      # 项目技能
  hooks/                       # 生命周期 hooks
  agents/                      # 项目 agent 定义
  plugins/                     # 项目插件
  cron/                        # workspace cron specs
```

Notes:

- 全局 provider 设置、全局设置和 MCP 设置存储在 `~/.cline/data/settings/`。
- 全局 workflows 从 `~/.cline/data/workflows/` 解析。
- 全局规则、hooks、技能、agents、插件和 cron specs 直接从 `~/.cline/` 解析。
- 为兼容旧安装，规则、hooks、插件和 workflows 也可能从 `~/Documents/Cline/` 发现。

## What Goes Where?

- 使用 **global (`~/.cline/`)** 存放本机 NBG CLI、IDE 和 SDK 共享的默认配置。
- 使用 **project (`.cline/`)** 存放应该随仓库共享的团队行为。

可以提交要与团队共享的 `.cline/` 文件，但不要把密钥提交到仓库。

## Configure Through the CLI

使用交互式配置 UI：

```bash
nbg config
```

From there, you can view/edit:

- Settings (global + workspace)
- Rules
- Skills
- Hooks

## Useful Configuration Commands

Use a custom configuration directory:

```bash
nbg --config /path/to/custom/config "你的任务"
```

Or via environment variable:

```bash
export CLINE_DATA_DIR=/custom/path/to/cline
nbg "你的任务"
```

排障时查看 CLI 日志：

```bash
nbg doctor log
```

## Environment Variables

| 变量 | 说明 |
|------|------|
| `CLINE_DATA_DIR` | 自定义数据目录（替代 `~/.cline/data/`） |
| `CLINE_HUB_ADDRESS` | 覆盖 hub 地址（默认：`127.0.0.1:25463`） |
| `CLINE_SESSION_BACKEND_MODE` | 强制 backend 模式（`local`、`hub`、`remote`、`auto`） |
| `CLINE_SANDBOX` | 启用沙箱模式 |
| `CLINE_SANDBOX_DATA_DIR` | 沙箱会话存储目录 |
| `CLINE_HOOKS_DIR` | 额外 hooks 目录 |
| `CLINE_COMMAND_PERMISSIONS` | 限制 shell 命令的 JSON 策略 |

### CLINE_DATA_DIR

```bash
export CLINE_DATA_DIR=/custom/path/to/cline
nbg "你的任务"
```

### CLINE_COMMAND_PERMISSIONS

限制 NBG 可以执行的 shell 命令：

```bash
export CLINE_COMMAND_PERMISSIONS='{"allow": ["npm *", "git *"], "deny": ["rm -rf *"]}'
```

格式：

```json
{
  "allow": ["pattern1", "pattern2"],
  "deny": ["pattern3"],
  "allowRedirects": true
}
```

规则：

- `deny` 覆盖 `allow`
- 设置 `allow` 后，不匹配 `allow` 的命令会被拒绝
- `allowRedirects` 控制 shell 重定向（`>`、`>>`、`<`），默认 `false`

## Security Notes

<Warning>
只使用来自可信来源的 rules、hooks、skills 和 plugins。
</Warning>

Hooks 和 plugins 可以执行代码。加入全局或项目配置前，应像审查可执行文件一样审查它们。
