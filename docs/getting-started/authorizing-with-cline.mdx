NBG 通过 **provider** 连接 AI 模型。当前保留 Cline 兼容 provider，同时默认支持 OpenAI-compatible、Anthropic、OpenAI、Google Gemini、OpenRouter 等 BYOK 路径。

- **OpenAI-compatible / BYOK**：使用自己的 provider 凭据，适合商业化部署和自有模型网关。
- **Cline 兼容 provider**：使用继承运行时中的 OAuth provider id `cline`。

## Menu

- [CLI Setup](#cli-setup)
- [IDE Setup](#ide-setup)

## CLI Setup

```bash
# 打开交互式认证设置
nbg auth

# 使用 Cline 兼容 provider OAuth
nbg auth cline

# 使用 OpenAI-compatible 或其他 BYOK provider
nbg auth --provider openai-compatible --apikey <key> --baseurl <url> --modelid <model>
```

无交互运行时，如果选择了 OAuth provider 但没有已保存凭据，`nbg` 会直接给出认证错误，不会隐藏式打开浏览器。

## IDE Setup

<Steps>
  <Step title="打开 NBG 设置">
    点击 NBG 面板中的设置图标。
  </Step>

  <Step title="选择 provider">
    从 **API Provider** 下拉框选择目标 provider。
  </Step>

  <Step title="配置认证">
    - **BYOK provider:** 在 **API Key** 字段粘贴你的 key。
    - **Cline 兼容 provider:** 点击 **Sign In** 并完成 OAuth。
    - **本地运行时（Ollama/LM Studio）:** 不需要 key，确认本地服务已运行。
  </Step>

  <Step title="选择模型">
    从 **Model** 下拉框选择模型。
  </Step>
</Steps>

## Provider Options

### BYOK 和 OpenAI-compatible

适合把 NBG 接入自有模型网关、云厂商或本地推理服务。生产环境优先把 key 放在本地配置或环境变量中，不要提交到仓库。

### Cline 兼容 Provider

- 继承运行时中的 provider id 是 `cline`
- 支持 OAuth 登录
- 用于兼容已有账号、配置和测试路径

NBG 不把旧 Dashboard 作为默认商业化入口。需要使用该兼容 provider 时，通过 `nbg auth cline` 或 IDE 登录入口完成认证。

## Troubleshooting

| Issue | Fix |
|-------|-----|
| "Unauthorized: Please sign in" | Session expired. Click **Sign In** to re-authenticate. |
| Browser doesn't open | Check default browser settings. Copy the URL from the NBG output panel manually. |
| Frequent re-authentication | Check org security policies. Ensure you're not clearing IDE secrets. Try a full sign-out/sign-in. |
| Can't access organization | Verify membership with the compatible provider account. Ask your admin about permissions. Sign out and back in. |
