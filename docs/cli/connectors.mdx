<Warning>
  连接器当前仅适用于 NBG CLI。
</Warning>

连接器让你从消息平台与 agent 对话。每条传入消息都会创建或继续一个 agent 会话，agent 响应会发送回同一会话。

## 设置向导

运行 `nbg connect` 打开交互向导，完成平台选择、凭据录入、安全配置和高级选项（provider、模型、系统提示词、agent 模式）。

```bash
nbg connect
```

## 支持平台

| 平台 | 直接命令 | 必需凭据 |
|------|----------|----------|
| Telegram | `nbg connect telegram` | Bot token |
| Slack | `nbg connect slack` | Bot token、签名密钥、公网 base URL |
| Google Chat | `nbg connect gchat` | Service account credentials JSON 或 ADC、公网 base URL |
| WhatsApp | `nbg connect whatsapp` | Phone number ID、access token、app secret、verify token、公网 base URL |
| Linear | `nbg connect linear` | API key 或 OAuth 凭据、webhook 签名密钥、公网 base URL |

## Telegram

<Steps>
  <Step title="创建 Telegram bot">
    打开 Telegram 并与 [@BotFather](https://t.me/BotFather) 对话。发送 `/newbot`，然后按提示操作：

    1. 输入显示名称，例如 `NBG`
    2. 输入以 `bot` 结尾的用户名，例如 `nbg_myname_bot`，必须在 Telegram 中唯一。
    3. BotFather 会返回 bot token，格式类似 `7123456789:AAH...`
  </Step>

  <Step title="启动连接器">
    ```bash
    nbg connect telegram -k <BOT-TOKEN>
    ```

    连接器会从 token 自动发现 bot 用户名。只有需要覆盖时才使用 `--bot-username`。
  </Step>

  <Step title="与 bot 对话">
    打开 Telegram，搜索你的 bot 用户名并发送消息。agent 会处理消息并在聊天中回复。
  </Step>
</Steps>

### 安全

默认情况下，任何找到 bot 的人都可以发送消息，并触发它在你的机器上执行任务。使用 `--hook-command` 限制访问。

<Steps>
  <Step title="获取 Telegram 用户 ID">
    在 Telegram 中给 [@userinfobot](https://t.me/userinfobot) 发消息，它会立即返回你的 user ID。
  </Step>

  <Step title="带访问控制启动">
    把 `12345` 替换成你的真实 Telegram user ID：

    ```bash
    nbg connect telegram -k <BOT-TOKEN> \
      --hook-command 'jq -r ".payload.actor.participantKey" | grep -q "telegram:id:12345" && echo "{\"action\":\"allow\"}" || echo "{\"action\":\"deny\",\"message\":\"unauthorized\"}"'
    ```
  </Step>
</Steps>

`--hook-command` 会通过 stdin 接收包含发送者信息的每条传入消息。脚本返回 `{"action": "allow"}` 或 `{"action": "deny", "message": "reason"}`。未设置 `--hook-command` 时，所有消息都会自动批准。

## Slack

需要 bot token、签名密钥和公网 base URL。

```bash
nbg connect slack --bot-token <BOT-TOKEN> --signing-secret <SECRET> --base-url <URL>
```

每个 Slack thread 会映射到一个 agent 会话，因此 agent 会在同一 thread 内保持上下文。

## Google Chat

需要 service account credentials JSON 文件和公网 base URL。

```bash
nbg connect gchat --credentials-json <JSON> --base-url <URL>
```

## WhatsApp

需要 phone number ID、access token、app secret、webhook verify token 和公网 base URL。

```bash
nbg connect whatsapp --phone-number-id <ID> --access-token <TOKEN> --app-secret <SECRET> --verify-token <VERIFY> --base-url <URL>
```

## Linear

需要 API key、webhook 签名密钥和公网 base URL。

```bash
nbg connect linear --api-key <KEY> --webhook-secret <SECRET> --base-url <URL>
```

## 管理连接器

```bash
# 停止所有连接器
nbg connect --stop

# 停止指定连接器
nbg connect telegram --stop
```

## Hook Command 协议

`--hook-command` 适用于所有连接器。脚本会通过 stdin 接收 JSON payload：

```json
{
  "payload": {
    "actor": {
      "participantKey": "telegram:id:12345",
      "displayName": "用户名"
    },
    "message": "传入消息文本"
  }
}
```

脚本返回 `{"action": "allow"}` 或 `{"action": "deny", "message": "reason"}`。返回 `deny` 时，连接器不会把该消息交给 agent。

## 同时运行多个连接器

多个连接器可以同时运行，并共享同一个 hub：

```bash
# 终端 1
nbg connect telegram -k $TELEGRAM_TOKEN

# 终端 2
nbg connect slack --bot-token $SLACK_TOKEN --signing-secret $SECRET --base-url $URL
```

连接器依赖 hub。如果没有自动启动，可以运行 `nbg hub start`。
