<Warning>
  定时任务当前适用于 NBG CLI、SDK 和 Kanban，暂不适用于 VSCode 与 JetBrains 扩展。
</Warning>

CLI 支持通过 hub 按 cron 计划运行 agent。定时 agent 会跨进程重启持久化，并独立于任何终端会话运行。

## 定时任务向导

运行 `nbg schedule` 打开交互菜单，用于创建和管理定时任务、浏览执行历史并查看性能统计。

```bash
nbg schedule
```

向导提供：

| 操作 | 说明 |
|------|------|
| 新建定时任务 | 使用 cron 和 prompt 创建重复任务 |
| 查看定时任务 | 查看所有任务的状态和下次运行时间 |
| 即将运行 | 预览接下来 10 次计划执行 |
| 活跃执行 | 查看当前正在运行的任务 |
| 立即触发 | 立即运行选中的定时任务 |
| 暂停 / 恢复 | 暂停或恢复定时任务 |
| 执行历史 | 查看历史运行的状态、耗时、token 和成本 |
| 统计信息 | 查看成功率、平均耗时和最近失败 |
| 删除 | 删除定时任务 |

## 使用参数创建定时任务

```bash
nbg schedule create "PR 摘要" \
  --cron "0 9 * * MON-FRI" \
  --prompt "列出所有打开的 PR 及其审查状态" \
  --workspace /path/to/repo \
  --model anthropic/claude-sonnet-4-6
```

## 管理定时任务

```bash
nbg schedule list
nbg schedule trigger <schedule-id>
nbg schedule pause <schedule-id>
nbg schedule resume <schedule-id>
nbg schedule delete <schedule-id>
nbg schedule history <schedule-id>
nbg schedule stats <schedule-id>
nbg schedule upcoming
```

## Cron 表达式参考

| 表达式 | 计划 |
|--------|------|
| `*/5 * * * *` | 每 5 分钟 |
| `*/15 * * * *` | 每 15 分钟 |
| `0 * * * *` | 每小时 |
| `0 */6 * * *` | 每 6 小时 |
| `0 0 * * *` | 每天午夜 |
| `0 9 * * *` | 每天上午 9 点 |
| `0 9 * * 1-5` | 每个工作日上午 9 点 |
| `0 9 * * 1` | 每周一上午 9 点 |
| `0 0 1 * *` | 每月第一天 |

## 示例

### 每日站会摘要

```bash
nbg schedule create "站会准备" \
  --cron "0 8 * * MON-FRI" \
  --prompt "总结：(1) 昨天合并的 PR，(2) 当前审查中的 PR，(3) 分配给团队成员的 open issues。" \
  --workspace /path/to/repo
```

### 每周依赖检查

```bash
nbg schedule create "依赖检查" \
  --cron "0 10 * * MON" \
  --prompt "检查过期 npm 依赖。若存在安全漏洞，创建更新分支并打开 PR。" \
  --workspace /path/to/project
```

### 代码库健康报告

```bash
nbg schedule create "代码健康检查" \
  --cron "0 6 * * MON" \
  --prompt "分析代码库：(1) 没有测试覆盖的文件，(2) 超过 30 天的 TODO/FIXME 注释，(3) 超过 100 行的函数。" \
  --workspace /path/to/project
```

## 路由执行结果

结合 [连接器](/cli/connectors) 可以把结果发送到消息平台：

```bash
nbg connect telegram -k $BOT_TOKEN

nbg schedule create "晨报" \
  --cron "0 8 * * *" \
  --prompt "总结仓库夜间活动"
```

定时任务依赖 hub。创建定时任务时 hub 会自动启动。
