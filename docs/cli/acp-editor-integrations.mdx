NBG CLI 支持 [Agent Client Protocol (ACP)](https://agentclientprotocol.com/)。ACP 是一个开放标准，让 AI coding agent 可以接入不同编辑器和 IDE。通过 ACP，你可以在常用开发环境中使用 NBG 的 Skills、Hooks 和 MCP 集成能力。

## 为什么使用 ACP？

- **编辑器灵活性**：在 JetBrains、Neovim、Zed 或任何 ACP-compatible editor 中使用 NBG
- **能力一致**：跨编辑器访问 NBG CLI 的核心 agent 能力
- **团队一致性**：不同开发工作流使用同一套 assistant 配置
- **开放标准**：基于 Zed 推出的 Agent Client Protocol specification

## JetBrains IDEs

[JetBrains](https://www.jetbrains.com) IDEs 包括 IntelliJ IDEA、PyCharm、WebStorm 等。内置 AI Assistant 支持 ACP。

<Note>
**推荐：原生 JetBrains 插件**

如果需要完整 IDE 集成，优先参考 NBG 安装指南中的原生插件路径。

下面的 ACP 配置是另一种在 JetBrains IDE 中使用 NBG CLI 能力的方式。
</Note>

### 配置

1. **安装 NBG CLI**：
   ```bash
   npm i -g nbg
   ```

2. **配置认证**：
   ```bash
   nbg auth
   ```

3. **配置 JetBrains AI Assistant**：
   - 打开 JetBrains IDE
   - 进入 `Settings | Tools | AI Assistant | Agents`
   - 点击 "Add Custom Agent"
   - 打开或创建 `~/.jetbrains/acp.json`

4. **把 NBG 加入 `acp.json`**：
   ```json
   {
     "agent_servers": {
       "NBG": {
         "command": "nbg",
         "args": ["--acp"],
         "env": {}
       }
     }
   }
   ```

5. **使用 NBG**：
   - 打开 AI Chat tool window
   - 在 agent dropdown 中选择 "NBG"
   - 开始在 JetBrains IDE 中使用 NBG

<Tip>
JetBrains AI Assistant 可以把内置 MCP server 暴露给 NBG，让 NBG 访问 IDE 专属工具和上下文。
</Tip>

## Neovim

[Neovim](https://neovim.io) 是可扩展的 Vim-based text editor。可以通过支持 ACP 的 [agentic.nvim](https://github.com/carlos-algms/agentic.nvim) 或 [avante.nvim](https://github.com/yetone/avante.nvim) 接入 NBG。

### 使用 agentic.nvim 配置

1. **安装 NBG CLI**：
   ```bash
   npm i -g nbg
   ```

2. **配置认证**：
   ```bash
   nbg auth
   ```

3. **使用 lazy.nvim 安装 agentic.nvim**：
   ```lua
   {
     "carlos-algms/agentic.nvim",
     opts = {
       provider = "nbg-acp",
       acp_providers = {
         ["nbg-acp"] = {
           command = "nbg",
           args = {"--acp"},
         },
       },
     },
     keys = {
       {"<C-\\>", function() require("agentic").toggle() end, mode={"n","v","i"}, desc="Toggle NBG Chat"},
     },
   }
   ```

4. **使用 NBG**：
   - 按 `<C-\>` 打开或关闭 NBG chat
   - 在 Neovim 中使用 NBG

### 使用 avante.nvim 配置

参考 [avante.nvim documentation](https://github.com/yetone/avante.nvim) 配置 external ACP agent，并指向 `nbg --acp`。

## Zed

[Zed](https://zed.dev) 是强调速度和协作的 code editor。Zed 团队创建了 Agent Client Protocol，因此可以自然接入 NBG CLI。

### 配置

1. **安装 NBG CLI**：
   ```bash
   npm i -g nbg
   ```

2. **配置认证**：
   ```bash
   nbg auth
   ```

3. **配置 Zed**：
   - 打开 Zed settings (`Cmd/Ctrl + ,`)
   - 把 NBG 加入 `settings.json`：
   ```json
   {
     "agent_servers": {
       "NBG": {
         "type": "custom",
         "command": "nbg",
         "args": ["--acp"],
         "env": {}
       }
     }
   }
   ```

4. **使用 NBG**：
   - 打开 AI assistant panel
   - 在 agent dropdown 中选择 "NBG"
   - 在 Zed 中使用 NBG

## 其他编辑器

任何支持 Agent Client Protocol 的编辑器都可以运行 NBG。查看对应编辑器的 ACP 配置说明，并指向：

```bash
nbg --acp
```

## 故障排查

### Agent 没有出现

- 确认已全局安装 NBG CLI：`npm i -g nbg`
- 确认认证已配置：`nbg auth`
- 确认 `nbg --acp` 可以正常运行
- 修改配置后重启编辑器

### 权限错误

如果 NBG 无法访问文件或运行命令：
- 确认编辑器的 ACP integration 传入了正确 working directory
- 检查项目文件权限
- 确认 NBG 的 approval settings 配置正确

### 连接问题

- 确认没有其他 NBG 实例占用同一配置目录
- 查看编辑器日志中的 ACP 相关错误
- 手动运行 `nbg --acp` 测试连接
