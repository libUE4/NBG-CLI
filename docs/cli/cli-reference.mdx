## 用法概览

```bash
nbg [options] [command] [prompt]
```

## 命令

### `nbg`（默认）

启动任务或进入交互模式。

```bash
nbg
nbg "你的提示词"
nbg "运行测试并修复失败"
echo "提示词" | nbg
```

### `auth [options] [provider]`

配置 AI provider 认证。

```bash
nbg auth
```

### `config [options]`

显示当前配置。

```bash
nbg config
```

### `connect [options] [channel]`

连接外部频道。详见 [连接器](/cli/connectors)。

```bash
nbg connect
nbg connect [channel]
```

### `mcp`

管理 MCP server。详见 [MCP](/mcp/mcp-overview)。

```bash
nbg mcp
```

### `doctor`

诊断并修复配置问题。

```bash
nbg doctor
```

### `history|h [options]`

查看会话历史或管理保存的会话。

```bash
nbg history
nbg h
```

### `hook`

从 stdin 处理 hook payload。

```bash
cat payload.json | nbg hook
```

### `plugin`

管理 NBG 插件。支持从文件 URL、npm、git 仓库或本地路径安装插件。详见 [插件](/customization/plugins)。

```bash
nbg plugin install <source>  # 安装插件
nbg plugin i <source>        # 简写别名
```

| 参数 | 说明 |
|--------|-------------|
| `--npm` | 将 source 参数视为 npm 包 |
| `--git` | 将 source 参数视为 git 仓库 |
| `--force` | 替换同一 source 参数的已有安装 |
| `--json` | 以 JSON 输出结果 |
| `--cwd <path>` | 安装到 `<path>/.cline/plugins`，而不是全局目录 |

可以用仓库内的 TypeScript LSP 插件示例验证安装：

```bash
nbg plugin install ./sdk/examples/plugins/typescript-lsp
```

### `schedule`

管理定时 agent。详见 [定时任务](/cli/scheduling)。

```bash
nbg schedule
```

### `hub`

管理本地 hub 守护进程。

```bash
nbg hub
```

### `update [options]`

检查并安装可用更新。

```bash
nbg update
```

### `version`

显示 NBG CLI 版本号。

```bash
nbg version
nbg -V
```

### `kanban`

启动 Kanban 看板应用并退出。

```bash
nbg kanban
```

### `help [command]`

显示指定命令的帮助。

```bash
nbg help doctor
```

## 环境变量

NBG 当前保留 Cline 运行时兼容层，因此这些变量名暂时仍使用 `CLINE_*`。

| 变量 | 说明 |
|----------|-------------|
| `CLINE_DATA_DIR` | 自定义配置目录（替代 `~/.cline/data/`） |
| `CLINE_HUB_ADDRESS` | 覆盖 hub 地址（默认：`127.0.0.1:25463`） |
| `CLINE_SESSION_BACKEND_MODE` | 强制后端模式（`local`、`hub`、`remote`、`auto`） |
| `CLINE_SANDBOX_DATA_DIR` | 沙箱会话存储目录 |
| `CLINE_SANDBOX` | 启用沙箱模式 |
| `CLINE_HOOKS_DIR` | 额外 hooks 目录 |
| `CLINE_BUILD_ENV` | 设置为 `development` 以启用调试功能 |
| `CLINE_DEBUG_PORT_BASE` | Node.js inspector 起始端口 |
| `CLINE_COMMAND_PERMISSIONS` | 限制 shell 命令的 JSON 策略（见下方） |

### CLINE_COMMAND_PERMISSIONS

限制 agent 可以执行的 shell 命令：

```bash
export CLINE_COMMAND_PERMISSIONS='{"allow": ["npm *", "git *"], "deny": ["rm -rf *", "sudo *"]}'
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `allow` | `string[]` | 允许执行的命令 glob。设置后只允许匹配命令。 |
| `deny` | `string[]` | 禁止执行的命令 glob。拒绝规则优先级最高。 |
| `allowRedirects` | `boolean` | 是否允许 shell 重定向（`>`、`>>`、`<`）。默认：`false`。 |

## JSON 输出格式

使用 `--json` 时，每条消息会作为独立 JSON 对象输出一行：

```json
{"type": "say", "text": "我现在创建文件。", "ts": 1760501486669, "say": "text"}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | `"ask"` 或 `"say"` | 消息类别 |
| `text` | `string` | 消息内容 |
| `ts` | `number` | Unix 毫秒时间戳 |
| `say` | `string` | `type` 为 `"say"` 时的子类型 |
| `ask` | `string` | `type` 为 `"ask"` 时的子类型 |
| `reasoning` | `string` | 模型推理内容（如果可用） |
| `partial` | `boolean` | 流式输出未完成时为 `true` |

## 配置文件

这些路径当前仍沿用 `.cline` 兼容目录。迁移到 `.nbg` 需要单独的兼容方案。

```
~/.cline/
  data/
    settings/
      providers.json             # API key 和 provider 配置
      rules/                     # 全局规则
      skills/                    # 全局技能
    teams/                       # 团队状态
    sessions/                    # 会话数据库（SQLite）
    logs/
      hub-daemon.log             # Hub 日志
  plugins/                       # 全局插件
    _installed/                  # 由 `nbg plugin install` 管理

.cline/                          # 项目根目录
  rules/                         # 项目规则
  skills/                        # 项目技能
  hooks/                         # 生命周期 hooks
  plugins/                       # 项目插件
  mcp.json                       # MCP server 配置
  agents.yaml                    # Agent 定义
```
