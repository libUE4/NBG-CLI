NBG 在聊天中提供 slash commands，用于管理对话、规划复杂实现和触发可复用技能。

<Tip>
**第一次使用 slash commands？** 可以观看这个 [quick video walkthrough](https://youtu.be/MxS5Jerpf-o) 了解命令效果。
</Tip>

## Slash Commands

在聊天输入框中输入 `/` 可以查看可用的 slash commands：

| 命令 | 作用 |
|---------|--------------|
| `/newtask` | 基于当前对话提炼上下文，开启一个新任务 |
| `/smol` | 压缩对话历史，同时保留关键上下文 |
| `/newrule` | 创建 rule 文件，让 NBG 记住你的偏好 |
| `/deep-planning` | 调研代码库、充分规划，然后创建实现任务 |
| `/explain-changes` | 为任意 git diff 生成 AI 解释（仅 VS Code） |
| `/reportbug` | 携带诊断信息报告 bug |

### /newtask

`/newtask` 类似一次开发交接。它会把总体计划、已完成工作、相关文件和下一步等关键信息打包到一个干净上下文的新任务中，同时丢弃工具调用和实现细节中的噪声。

处理复杂实现时可以使用 `/newtask`。例如一个 10 步任务已经完成 3 步，且上下文窗口已经接近上限时，使用 `/newtask` 可以保留关键决策、文件变更和进度。

### /smol

`/smol`（或别名 `/compact`）会在保留关键上下文的同时压缩对话历史。它不会像 `/newtask` 一样创建新任务，而是把当前对话压缩为完整摘要，让你继续留在同一个任务中工作。

当你正在深入调试或头脑风暴，并希望在不丢失已有结论的情况下继续当前任务时，使用 `/smol`。更多细节见 [Smol Command](#smol)。

### /newrule

`/newrule` 会创建一个 rule 文件，用于让 NBG 记住你的偏好。NBG 会引导你设置沟通风格、编码规范、项目上下文和可复用实践。该 rule 会保存到兼容的 `.clinerules` 目录，并在后续对话中自动加载。

如果你发现自己在多个任务中反复说明同样要求，可以使用 `/newrule`。更多规则说明见 [NBG Rules](/customization/cline-rules)。

### /deep-planning

`/deep-planning` 会让 NBG 先像架构师一样调研代码库、提出澄清问题，并在写代码前创建完整实现计划。Deep planning 通常包含四步：

1. **Silent Investigation** - NBG 调研代码库结构和既有模式
2. **Discussion** - 围绕需求和方案提出针对性问题
3. **Plan Creation** - 生成包含详细规格的 `implementation_plan.md`
4. **Task Creation** - 创建带有可跟踪实现步骤的新任务

当功能会影响代码库多个部分、涉及架构调整或复杂集成时，适合使用 `/deep-planning`。

### /explain-changes

<Note>
该命令仅在 VS Code 中可用。
</Note>

`/explain-changes` 会为任意 git diff 生成 AI 解释。你可以解释最近一次 commit、未提交变更、staged changes、指定 commits、分支、PR，或任意变更范围。

在 review 代码、熟悉新代码库或理解变更时可以使用 `/explain-changes`。完整用例和示例见 [Explain Changes Command](#explain-changes)。

### /reportbug

`/reportbug` 会收集诊断信息，帮助你报告 NBG 问题。它会汇总配置、近期错误和系统信息等上下文，让 bug report 更容易定位。

遇到异常行为、崩溃或需要反馈的问题时，可以使用 `/reportbug`。

## 通过 Slash Commands 使用 Skills

除了内置命令，你也可以直接在聊天中用 slash commands 触发已启用的 skills。

- 输入 `/` 打开命令建议。
- 选择一个 skill 命令，例如 `/aws-deploy`。
- NBG 会加载该 skill，并把它的 `SKILL.md` 指令应用到当前任务。

任何已启用的 skill 都可以用这种方式触发。这样可以快速复用 skill 专属指导，不必每次重复输入同样说明。

设置和管理细节见 [Skills](/customization/skills#triggering-skills-with-slash-commands)。
