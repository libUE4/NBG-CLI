Tools 是模型在执行过程中可以调用的函数。每个 tool 包含 name、description 和 schema，SDK 会把这些信息发送给模型。随后模型可以指示 SDK 执行 tool function，并把结果送回对话。

## 内置工具

`ClineCore` 可以启用内置 tool suite：

| Tool | 说明 |
|------|-------------|
| `read_files` | 读取一个或多个文件 |
| `search_codebase` | 搜索 workspace |
| `run_commands` | 执行 shell commands |
| `fetch_web_content` | 获取 web content |
| `apply_patch` | 应用 patch/diff edits |
| `editor` | 编辑文件 |
| `skills` | 调用已配置 skills |
| `ask_question` | 向用户请求输入 |
| `submit_and_exit` | 提交最终答案并停止 |

<Note>
  如果需要更强控制，可以直接使用 `Agents` package。它默认不包含内置工具，构造 agent 时只传入你需要的 tools。
</Note>

## 使用 createTool 创建 Custom Tools

SDK 的核心能力之一是创建和注册 custom tools。因为 tools 由代码实现，而不是只靠 prompts，因此可以添加可共享、上下文高效且行为更确定的能力。

### 快速示例

将 `createTool` 与 zod schema 一起使用，可以创建 type-safe tools：

```typescript
import { createTool } from "@cline/sdk"
import { z } from "zod"

const searchDatabase = createTool({
  name: "search_database",
  description: "Search the application database. Returns matching records as JSON.",
  inputSchema: z.object({
    query: z.string().describe("Search query"),
    limit: z.number().optional().describe("Maximum results to return"),
  }),
  async execute(input) {
    const results = await db.search(input.query, input.limit ?? 10)
    return { results, count: results.length }
  },
})
```

`createTool` 也支持 raw JSON Schema：

```typescript
const searchDatabase = createTool({
  name: "search_database",
  description: "Search the application database.",
  inputSchema: {
    type: "object",
    properties: {
      query: { type: "string", description: "Search query" },
      limit: { type: "number", description: "Maximum results to return" },
    },
    required: ["query"],
  },
  async execute(input) {
    return await db.search(input.query, input.limit ?? 10)
  },
})
```

完整 tools 示例见 [cli-agent](https://github.com/libUE4/NBG-CLI/tree/main/sdk/apps/examples/cli-agent)（shell tool）和 [code-review-bot](https://github.com/libUE4/NBG-CLI/tree/main/sdk/apps/examples/code-review-bot)（包含多个 tools 与 completion lifecycle）。

完整教程见 [Creating Custom Tools](/sdk/guides/creating-custom-tools)。精确类型见 [Tools API](/sdk/reference/tools-api)。

## 注册 Tools

使用 `ClineCore` 时，可以通过 session config 中的 `extraTools` 传入 custom tools：

```typescript
await cline.start({
  prompt: "Analyze customer records",
  config: {
    // ...
    extraTools: [searchDatabase],
  },
})
```

使用 `Agent` 时，在 constructor 中传入全部 tools：

```typescript
const agent = new Agent({
  tools: [searchDatabase, myOtherTool],
  // ...
})
```

通过 plugins/extensions 更容易共享 tools：

```typescript
const plugin: AgentPlugin = {
  name: "database-tools",
  manifest: { capabilities: ["tools"] },
  setup(api) {
    api.registerTool(searchDatabase)
  },
}
```
更多信息见 [Plugins](/sdk/plugins)。

## Tool Policies

可以通过 Tool Policies 控制 tools 的使用方式，包括 tool 是否可见以及是否需要 approval。

```typescript
const agent = new Agent({
  tools: [readTool, writeTool, deleteTool],
  toolPolicies: {
    read_data: { autoApprove: true },
    write_data: { autoApprove: false },
    delete_data: { enabled: false },
  },
  // ...
})
```

| Policy | Effect |
|--------|--------|
| `{ autoApprove: true }` | Run without asking |
| `{ autoApprove: false }` | Ask before running |
| `{ enabled: false }` | Hide/disable the tool |

Tool names not listed in `toolPolicies` default to enabled and auto-approved.

## MCP Tools

Tools work along side MCP. `ClineCore` can load MCP settings through its runtime/config extension path. MCP tools are registered alongside built-in and custom tools when MCP support is enabled for the session.
