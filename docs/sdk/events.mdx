SDK 暴露两个相关 event surfaces：

- `@cline/agents` 中的 `AgentRuntimeEvent`，通过 `agent.subscribe(listener)` 传递。
- `@cline/core` 中的 `AgentEvent` / `CoreSessionEvent`，通过 core session adapters 和 `cline.subscribe(listener)` 传递。

本页说明 event handling patterns。Event shapes 见 [Events reference](/sdk/reference/events)。

## AgentRuntime Events

```typescript
const unsubscribe = agent.subscribe((event) => {
  if (event.type === "assistant-text-delta") {
    process.stdout.write(event.text ?? "")
  }
})

await agent.run("解释这个代码库")
unsubscribe()
```

`AgentRuntimeEvent` 是 browser-compatible runtime 发出的 low-level event stream。

## Core / Host-Facing Agent Events

`AgentEvent` 是 `@cline/core` session orchestration 使用的 host-facing event shape。直接使用 `Agent` 时，优先使用 `agent.subscribe(...)` 和 `AgentRuntimeEvent`。

常见 host-facing event categories：

| Category | Events | 用途 |
|----------|--------|---------|
| Content | `content_start`, `content_update`, `content_end` | Text/reasoning/tool UI |
| Iterations | `iteration_start`, `iteration_end` | Progress tracking |
| Usage | `usage` | Token/cost tracking |
| Notices | `notice` | Recovery and status updates |
| Completion | `done`, `error` | Final state and failure handling |

## ClineCore Session Events

使用 `cline.subscribe()` 订阅 session-level events：

```typescript
const unsubscribe = cline.subscribe((event) => {
  console.log(event.type, event.sessionId)
})

await cline.start({ /* ... */ })
unsubscribe()
```

只需要某个 session 时，传入 `sessionId` filter：

```typescript
const unsubscribe = cline.subscribe(listener, { sessionId })
```

## Streaming UI Pattern

订阅 events 可以构建 real-time UIs：

```typescript
agent.subscribe((event) => {
  switch (event.type) {
    case "assistant-text-delta":
      onUpdate({ type: "text", content: event.text ?? "" })
      break
    case "tool-started":
      onUpdate({ type: "tool_start", content: event.toolCall.toolName })
      break
    case "tool-finished":
      onUpdate({ type: "tool_end", content: event.toolCall.toolName })
      break
    case "run-finished":
      onUpdate({ type: "done", content: event.result.status })
      break
  }
})
```

通过 SSE 将 agent events 流式发送到浏览器的完整示例见 [multi-agent example](https://github.com/libUE4/NBG-CLI/tree/main/sdk/apps/examples/multi-agent)。它会并行启动多个 agents，并把每个 agent 的 events 流式发送到独立 UI cards。

## Usage Tracking Pattern

```typescript
let totalTokens = 0

agent.subscribe((event) => {
  if (event.type === "usage-updated") {
    totalTokens = event.usage.inputTokens + event.usage.outputTokens
  }
})
```

## Runtime State Snapshots

需要读取当前状态时调用 `snapshot()`：

```typescript
const snapshot = agent.snapshot()
console.log(snapshot.status, snapshot.iteration, snapshot.usage)
```
