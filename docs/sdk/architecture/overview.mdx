SDK 拆分为分层 packages。依赖只向下流动：`core` 依赖 `agents`、`llms` 和 `shared`；`agents` 依赖 `llms` 和 `shared`；`llms` 依赖 `shared`。

## Package Stack

```txt
Your application / NBG CLI / VS Code / JetBrains
          │
          ▼
@cline/core
Sessions, storage, built-in tools, hub, automation, telemetry
          │
          ├── @cline/agents
          │   Browser-compatible AgentRuntime / Agent loop
          │
          ├── @cline/llms
          │   Provider handlers, gateway, model catalogs
          │
          └── @cline/shared
              Types, schemas, tools, hooks, extension contracts
```

## Packages

### @cline/core

Node runtime/orchestration layer。

主要 exports 包括：

| Export | 说明 |
|--------|-------------|
| `ClineCore` | 主要 runtime entry point |
| `ClineCoreOptions` | Constructor options |
| `ClineCoreStartInput` | Session start input |
| `CoreSessionConfig` | Session configuration |
| `SessionRecord` | 持久化 session metadata |
| `AgentPlugin` | Public plugin type |
| `createTool` | 从 shared re-export |

能力包括：

- local/hub/remote runtime backends
- session manifests and message artifacts
- built-in tools
- tool approvals
- automation/scheduling services
- telemetry hooks
- plugin/extension loading
- team/sub-agent tools

依赖：`@cline/shared`、`@cline/llms`、`@cline/agents`。

### @cline/agents

Browser-compatible agent execution loop。

主要 exports 包括：

| Export | 说明 |
|--------|-------------|
| `AgentRuntime` | Core runtime class |
| `Agent` | `AgentRuntime` 的 alias |
| `createAgentRuntime`, `createAgent` | Factory functions |
| `AgentRuntimeConfig` | Constructor config union |
| `AgentRunInput`, `AgentEventListener` | Runtime helper types |
| `createTool` | 从 `@cline/shared` re-export |

`AgentRuntime` methods 包括 `run`、`continue`、`abort`、`subscribe`、`restore` 和 `snapshot`。

依赖：`@cline/shared`、`@cline/llms`。

### @cline/llms

Provider 和 model layer。

主要 exports 包括：

| Export | 说明 |
|--------|-------------|
| `DefaultGateway`, `createGateway` | 创建 provider-backed agent models 的 gateway |
| `createHandler`, `createHandlerAsync` | Provider handler factories |
| `getAllProviders`, `getProviderIds`, `getModelsForProvider` | Catalog helpers |
| `registerProvider`, `registerModel` | Runtime registry extension |
| `ModelInfo`, `ProviderInfo` | Provider/model metadata |

依赖：`@cline/shared`。

### @cline/shared

用于 shared contracts 和 utilities 的 foundation package。

主要 exports 包括：

| Export | 说明 |
|--------|-------------|
| `createTool` | 创建 typed tools 的 helper |
| `AgentTool`, `AgentToolContext`, `ToolPolicy` | Tool interfaces |
| `AgentEvent`, `AgentResult`, `AgentConfig` | Host-facing agent types |
| `AgentRuntimeEvent`, `AgentRunResult` | Runtime-facing agent types |
| `HookEngine`, `HookStage`, `HookPolicies` | Hook contracts 和 engine |
| `ContributionRegistry`, `AgentExtensionApi` | Extension registration contracts |
| `ModelInfo`, `Message`, `ContentBlock` | Model/message types |
| `BasicLogger`, `noopBasicLogger` | Logging contracts |

没有 higher-layer dependencies。

## Install

```bash
npm install @cline/sdk
```

`@cline/sdk` 会 re-export `@cline/core` 的全部内容。只有在需要 lower-level control 时，才直接安装 `@cline/agents` 或 `@cline/llms`。

## Design Principles

### Strict dependency direction

依赖只能向下流动。Lower layers 可以保持 embeddable，不会拉入完整 runtime。

### Browser-compatible agent loop

`@cline/agents` 暴露 browser-compatible runtime。它不负责 session storage、built-in file/shell tools、hub transports 或 Node-specific orchestration。

### Core as orchestration layer

`@cline/core` 负责 Node runtime integration：session persistence、built-in tools、automation、hub/remote transports、telemetry 和 extension loading。
