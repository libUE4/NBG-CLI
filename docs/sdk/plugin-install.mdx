NBG CLI 提供 `plugin install` 命令，可从多种来源安装 plugins。你也可以通过 session config 以编程方式加载 plugins。

## CLI Installation

使用 `nbg plugin install` 命令从 file URLs、npm、git repositories 或 local paths 安装 plugins。

```sh
nbg plugin install <source> [options]
```

也可以使用 shorthand `nbg plugin i`。

### Install from a File URL

```sh
nbg plugin install https://github.com/libUE4/NBG-CLI/blob/main/sdk/examples/plugins/weather-metrics.ts
nbg plugin install https://raw.githubusercontent.com/libUE4/NBG-CLI/main/sdk/examples/plugins/weather-metrics.ts
```

从 HTTPS URL 安装单个 `.ts` 或 `.js` plugin file。GitHub `blob` URLs 会自动规范化为 raw file downloads。

### Install from npm

```sh
nbg plugin install --npm @scope/plugin-name
```

从 npm registry 安装 package 到 NBG plugin directory。

### Install from Git

```sh
nbg plugin install --git https://github.com/owner/repo.git
nbg plugin install --git github.com/owner/repo
```

浅 clone repository，并使用 blobless filter 提升速度，然后安装 dependencies。最终安装结果会排除 `.git` 目录。

### Install from Local Source

```sh
nbg plugin install ./path/to/plugin.ts     # Single file
nbg plugin install /absolute/path/to/dir   # Directory with package.json
```

支持 files 和 directories。本地安装会复制 source，过滤 `.git` 与 `node_modules`，并运行 `npm install` 解析 dependencies。

### Options

| Option | 说明 |
|--------|-------------|
| `--npm` | 将 source 视为 npm package name |
| `--git` | 将 source 视为 git repository URL |
| `--force` | 覆盖相同路径下的 existing plugin |
| `--json` | 以 JSON 输出结果，便于 scripting |
| `--cwd` | 安装到 `<path>/.cline/plugins`，并从 `<path>` 解析相对 local paths |

### Examples

```sh
# Install a local plugin directory
nbg plugin install ./my-custom-plugin

# Install a single plugin file from GitHub
nbg plugin install https://github.com/libUE4/NBG-CLI/blob/main/sdk/examples/plugins/weather-metrics.ts

# Install an npm package as a plugin
nbg plugin install --npm @cline/plugin-sql

# Install from a GitHub repo
nbg plugin install --git https://github.com/my-org/my-plugin.git

# Force reinstall an existing plugin
nbg plugin install --force --npm @scope/plugin-name

# Use shorthand
nbg plugin i --git github.com/owner/repo
```

### 列出已安装 Plugins

已安装 plugins 会出现在 CLI config 的 `plugins` section 中：

```sh
nbg config
```

### 从文件系统查找 Plugins

Project-scoped plugins 位于 workspace root 下兼容的 `.cline/plugins/` folder。在该项目中运行 NBG 时，CLI 会从这个目录自动发现 plugins。

```text
your-project/
├── .cline/
│   └── plugins/
│       ├── my-plugin.ts
│       └── another-plugin/
└── ...
```

要向项目添加 single-file plugin，可配合 `--cwd` 安装：

```sh
nbg plugin install https://github.com/owner/repo/blob/main/plugins/my-plugin.ts --cwd .
```

Global plugins 会从兼容路径 `~/.cline/plugins/` 发现，NBG 也可能从系统 Plugins folder 发现 plugins。希望 plugin 留在特定项目中时，使用 `--cwd .`。

## Programmatic Installation

### Using `pluginPaths` (ClineCore)

在 SDK code 中，可以使用 `pluginPaths` session config option 从 file paths 加载 plugins：

```typescript
import { ClineCore } from "@cline/sdk"

const cline = await ClineCore.create({ clientName: "my-app" })

await cline.start({
  config: {
    systemPrompt: "Analyze this project",
    // ...model/runtime config
    pluginPaths: ["/absolute/path/to/plugin.ts"],
  },
})
```

Plugin files 必须 default export 一个 `AgentPlugin`。

### Using `plugins` (Agent Runtime)

直接使用 `Agent` 或 `AgentRuntime` 时，在 `plugins` array 中传入 plugin instances：

```typescript
import { Agent } from "@cline/sdk"
import { myPlugin } from "./my-plugin"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  plugins: [myPlugin],
})
```

### Using `extensions` (ClineCore)

使用 `ClineCore` 时，改用 `extensions` config array：

```typescript
import { ClineCore } from "@cline/sdk"

const cline = await ClineCore.create({ clientName: "my-app" })

await cline.start({
  config: {
    systemPrompt: "Analyze customer records",
    extensions: [myPlugin],
  },
})
```
