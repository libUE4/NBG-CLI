`ClineCore` 是 NBG SDK 兼容层中的完整 programmable runtime。它提供文件、shell、search 和 web 等 built-in tools，也包含 session persistence、message history、tool approval callbacks、local/hub/remote backends、scheduling、automation APIs 和 plugin support。

在底层，`ClineCore` 使用 `@cline/agents` 中的 `Agent` class。如果想跳过完整 harness，并自行接入 tools、persistence 和 lifecycle，可以直接使用 `Agent`。

## 何时使用哪个

| 需求 | 使用 |
|------|-----|
| Sessions、persistence、message history | `ClineCore` |
| 面向 files、shell、search、web fetch 的 built-in tools | `ClineCore` |
| Hub/remote runtime | `ClineCore` |
| Scheduling 或 event automation | `ClineCore` |
| Multi-agent teams | `ClineCore` |
| Browser-compatible 或 lightweight in-process agent | `Agent` |
| 只需要 custom tools，不需要 built-ins | `Agent` |
| 需要完全控制 persistence 和 lifecycle | `Agent` |

## ClineCore

`ClineCore` 在 runtime execution 外包装了 application features：

- session manifests 和 message artifacts
- built-in tools
- tool approval callbacks
- local、hub 和 remote backends
- automation/scheduling APIs
- 可选 plugin paths 和 extensions

```typescript
import { ClineCore } from "@cline/sdk"

const cline = await ClineCore.create({
  clientName: "my-app",
  backendMode: "auto",
})

const session = await cline.start({
  prompt: "Set up GitHub Actions for this repo",
  config: {
    providerId: "anthropic",
    modelId: "claude-sonnet-4-6",
    apiKey: process.env.ANTHROPIC_API_KEY,
    systemPrompt: "You are a helpful coding assistant.",
    cwd: "/path/to/project",
    workspaceRoot: "/path/to/project",
    enableTools: true,
    enableSpawnAgent: false,
    enableAgentTeams: false,
  },
})

console.log(session.sessionId)
console.log(session.result?.finishReason)
```

### Methods

| Method | 用途 |
|--------|---------|
| `ClineCore.create(options)` | 创建 runtime |
| `start(input)` | 启动 session |
| `send({ sessionId, prompt })` | 发送 follow-up message |
| `subscribe(listener, options?)` | 监听 session events |
| `list(limit?, options?)` | 列出带 history metadata 的 sessions |
| `get(sessionId)` | 读取 session metadata |
| `readMessages(sessionId)` | 读取 session messages |
| `getAccumulatedUsage(sessionId)` | 读取 session token/cost totals |
| `abort(sessionId, reason?)` | 中止当前工作 |
| `stop(sessionId)` | 停止 session |
| `delete(sessionId)` | 删除 session |
| `dispose(reason?)` | 清理 runtime resources |

精确 signatures 见 [ClineCore reference](/sdk/reference/cline-core)。

## Backend Modes

| Mode | 行为 |
|------|----------|
| `auto` | 优先使用兼容 local hub，否则使用 local execution |
| `hub` | 要求兼容 WebSocket hub |
| `remote` | 连接已配置的 remote hub |
| `local` | 始终使用 local in-process execution 和 local storage |

Process topology 见 [Hub & Spoke](/sdk/architecture/hub-spoke)。

## Session Artifacts

`ClineCore` 会以文件形式存储 session manifests 和 messages。Session result 包含：

| Field | 含义 |
|-------|---------|
| `sessionId` | Session identifier |
| `manifest` | 已解析的 session manifest |
| `manifestPath` | Session manifest JSON 路径 |
| `messagesPath` | 持久化 message JSON 路径 |
| `result` | 可用时的最终 `AgentResult` |

## Tool Approval

简单场景可以使用 tool policies：

```typescript
await cline.start({
  prompt: "Audit this repo",
  config: {
    providerId: "anthropic",
    modelId: "claude-sonnet-4-6",
    apiKey: process.env.ANTHROPIC_API_KEY,
    systemPrompt: "You are a helpful coding assistant.",
    cwd: process.cwd(),
    workspaceRoot: process.cwd(),
    enableTools: true,
    enableSpawnAgent: false,
    enableAgentTeams: false,
  },
  toolPolicies: {
    read_files: { autoApprove: true },
    search_codebase: { autoApprove: true },
    run_commands: { autoApprove: false },
    editor: { autoApprove: false },
  },
})
```

当 application 需要动态决策时，使用 `requestToolApproval`：

```typescript
const cline = await ClineCore.create({
  clientName: "my-app",
  capabilities: {
    requestToolApproval: async (request) => {
      return { approved: request.toolName !== "run_commands" }
    },
  },
})
```

Tool behavior 和 policies 见 [Tools](/sdk/tools)。

## 直接使用 Agent

`Agent`（也导出为 `AgentRuntime`）是 `ClineCore` 构建其上的 stateless primitive。需要完全控制或不需要完整 harness 时，可以直接使用它。

`Agent` 是 `AgentRuntime` 的 alias。从 provider/model IDs 构造时使用 `Agent`；提供预构建 `AgentModel` 时使用 `AgentRuntime`。

`Agent` 运行 core loop：

```txt
run() or continue()
  -> model request
  -> tool calls, if any
  -> tool results
  -> repeat until complete
```

```typescript
import { Agent } from "@cline/sdk"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  apiKey: process.env.ANTHROPIC_API_KEY,
  systemPrompt: "You are a helpful coding assistant.",
  tools: [myTool],
})

agent.subscribe((event) => {
  if (event.type === "assistant-text-delta") {
    process.stdout.write(event.text ?? "")
  }
})

const result = await agent.run("Review this diff")
console.log(result.outputText)
```

### Agent Methods

| Method | 用途 |
|--------|---------|
| `run(input)` | 使用用户输入启动 run |
| `continue(input?)` | 使用可选新输入继续 |
| `abort(reason?)` | 中止 active run |
| `subscribe(listener)` | 监听 `AgentRuntimeEvent` events |
| `restore(messages)` | 替换 conversation history |
| `snapshot()` | 读取 runtime state |

精确 signatures 见 [Agent reference](/sdk/reference/agent)。

### Multi-Turn Conversations

`AgentRuntime` 会在内部保留 message state。第一次 run 后使用 `continue()`：

```typescript
await agent.run("What does this project do?")
await agent.continue("Now identify risky files")
```

如果在外部持久化 messages，继续前先用 `restore(messages)` 恢复。
