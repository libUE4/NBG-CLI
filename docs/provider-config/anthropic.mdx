## 选择 Anthropic 接入方式

- **[Anthropic API (key-based)](#getting-an-api-key)：** 使用 Anthropic API key。
- **[Claude Code (subscription-based)](#claude-code-subscription)：** 通过 Claude CLI 使用 Claude Max/Pro 订阅。

## Anthropic API (key-based)

### 获取 API key

1.  **注册或登录：** 打开 [Anthropic Console](https://console.anthropic.com/)，创建账号或登录现有账号。
2.  **进入 API Keys：** 打开 [API keys](https://console.anthropic.com/settings/keys) 页面。
3.  **创建密钥：** 点击 "Create Key"，使用易识别的名称，例如 "NBG"。
4.  **复制密钥：** **重要：** 立即复制 API key。离开页面后通常无法再次查看，请妥善保存。

### 在 NBG 中配置

> 参考 [认证与模型选择](/getting-started/authorizing-with-cline#menu)。

1.  **打开 NBG 设置：** 在 NBG 面板中点击设置图标 (⚙️)。
2.  **选择 provider：** 在 "API Provider" 下拉菜单中选择 "Anthropic"。
3.  **填写 API key：** 将 Anthropic API key 粘贴到 "Anthropic API Key" 字段。
4.  **选择模型：** 在 "Model" 下拉菜单中选择需要使用的 Claude 模型。
5.  **可选，自定义 Base URL：** 如需使用 Anthropic API 的自定义 base URL，勾选 "Use custom base URL" 并填写地址。大多数用户无需修改。

## Claude Code (Subscription)

如果你已经拥有 Claude Max/Pro，并希望在 NBG 中通过订阅方式访问 Claude，可使用该路径。

1. 按 Anthropic 文档安装并认证 Claude CLI：[Claude Code setup](https://docs.anthropic.com/en/docs/claude-code/setup)。
2. 在 NBG 设置中选择 **Claude Code** provider。
3. 设置 Claude CLI 路径。如果 `claude` 已在 PATH 中，通常直接填写 `claude`。

查找 Claude 路径：
- macOS / Linux / WSL / Git Bash: `which claude`
- Windows Command Prompt: `where claude`

说明：
- 使用 Claude 订阅额度，而不是 Anthropic API token 计费。
- 响应可能不会按 token 逐步流式返回。
- 此模式下图片上传和 prompt caching 能力有限。

### Extended Thinking

Anthropic 模型提供 "Extended Thinking" 功能，用于增强复杂任务中的推理能力。该功能会返回 `thinking` 内容块或推理摘要，并在最终回答前提供更多分析信号。

在 NBG 中启用 extended thinking 后，模型会生成 `thinking` 内容块，并将其用于最终回答。选择 Claude 模型后，可在模型选择菜单下方勾选 `Enable Extended Thinking`。

**关键点：**

-   **Summarized Thinking (Claude 3.7+)：** Claude 3.7+ 模型会返回完整 thinking 的摘要，以平衡可观察性、效率和滥用风险。计费通常按完整 thinking token 计算，而不只按摘要计算。
-   **Streaming：** Extended thinking 响应，包括 `thinking` blocks，可进行流式返回。
-   **Tool Use 与 Prompt Caching：** Extended thinking 会影响 tool use 和 prompt caching，例如需要回传 thinking blocks，以及涉及上下文和缓存失效规则。

完整机制、API 示例、tool use、prompt caching 和价格说明，请参考 [Anthropic Extended Thinking 官方文档](https://docs.anthropic.com/en/docs/build-with-claude/extended-thinking)。

### 提示

-   **Prompt Caching：** Claude 3 模型支持 [prompt caching](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching)，可降低重复 prompt 的成本和延迟。
-   **上下文窗口：** Claude 模型提供较大的上下文窗口，便于在 prompt 中包含更多代码和上下文。
-   **价格：** 最新价格请参考 [Anthropic Pricing](https://www.anthropic.com/pricing)。
-   **速率限制：** Anthropic 根据 [usage tiers](https://docs.anthropic.com/en/api/rate-limits#requirements-to-advance-tier) 设置速率限制。如果频繁触发限制，可联系 Anthropic sales，或通过 [OpenRouter](/provider-config/openrouter)、[Requesty](/provider-config/other-30-plus-providers#requesty) 等 provider 访问 Claude。
