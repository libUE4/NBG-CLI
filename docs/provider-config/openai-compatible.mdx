---
title: "OpenAI Compatible"
description: "在 NBG 中配置支持 OpenAI-compatible API 的模型 provider。"
---

NBG 支持多种兼容 OpenAI API 标准的模型 provider。你可以在不使用官方 OpenAI API 的情况下，继续通过熟悉的接口访问其他 provider 的模型，例如：

-   **本地模型：** 通过 Ollama、LM Studio 等工具运行的模型。
-   **云端 provider：** Perplexity、Together AI、Anyscale 等。
-   **其他 provider：** 任意提供 OpenAI-compatible API endpoint 的服务。

本文重点说明官方 OpenAI API 之外的 provider。官方 OpenAI API 请参考 [OpenAI 配置页](/provider-config/openai)。

### 通用配置

在 NBG 中使用 OpenAI-compatible provider，核心是配置以下字段：

1.  **Base URL：** provider 的 API endpoint。它通常不是 `https://api.openai.com/v1`，该地址用于官方 OpenAI API。
2.  **API Key：** 从目标 provider 获取的密钥，或使用 **Use Azure Identity Authentication**。
3.  **Model ID：** 需要调用的模型名称或标识符。

在 NBG 设置面板中点击 ⚙️ 图标后，可配置：

-   **API Provider：** 选择 "OpenAI Compatible"。
-   **Base URL：** 填写目标 provider 提供的 base URL。**这是关键步骤。**
-   **API Key：** 填写 provider API key。
-   **Model：** 选择或输入 model ID。
-   **Use Azure Identity Authentication：** 勾选后使用已有的 Azure managed identity 认证。该选项不会触发新的登录流程，而是复用已有状态，例如 `az login`。
-   **Model Configuration：** 可自定义模型高级参数，例如：
    -   Max Output Tokens
    -   Context Window size
    -   Image Support capabilities
    -   Computer Use (e.g., for models with tool/function calling)
    -   Input Price (per token/million tokens)
    -   Output Price (per token/million tokens)

**注意：** 不同 OpenAI-compatible provider 的 model ID 可能不同。请以目标 provider 的文档为准，确认支持的模型名称和特殊配置要求。

### 在 NBG 中使用 v0 (Vercel SDK)

-   v0 的 [AI SDK documentation](https://vercel.com/docs/v0/cline) 提供了多模型集成示例，其中不少模型兼容 OpenAI API。该外部原始文档可作为 NBG OpenAI-compatible 配置的参考资料。

-   v0 可通过 NBG 的 OpenAI Compatible provider 使用。

### Quickstart

1. 选择 OpenAI Compatible provider 后，将 Base URL 设为 `https://api.v0.dev/v1`。
2. 粘贴 v0 API key。
3. 设置 Model ID：`v0-1.0-md`。
4. 点击 Verify 确认连接。

### 排障

-   **"Invalid API Key"：** 检查 API key 是否填写正确，并确认它属于当前 provider。
-   **"Model Not Found"：** 确认 model ID 有效，并且在指定 Base URL 下可用。
-   **连接错误：** 检查 Base URL 是否正确、provider API 是否可从本机访问，以及网络、防火墙或代理是否正常。
-   **结果异常：** 尝试切换模型，或复核全部配置参数。

通过 OpenAI-compatible provider，NBG 可以接入更多模型服务。具体模型、价格和限制请始终以对应 provider 的官方文档为准。
