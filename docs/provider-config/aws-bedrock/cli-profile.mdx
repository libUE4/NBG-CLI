### 概览

NBG 支持通过 AWS credentials 或 AWS profile 访问 AWS Bedrock。对于企业和团队环境，优先建议使用 SSO/Federated roles，而不是长期 IAM credentials。

---

### 配置步骤

1. 安装 [latest version](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html) of AWS CLI。

    - 按 AWS 文档安装适合当前操作系统的 AWS CLI。

2. 使用 AWS CLI [配置 IAM authentication](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-sso.html)。

    - 如果尚未通过 IAM Identity Center 获得 AWS 访问权限，请按 [IAM User Guide](https://docs.aws.amazon.com/singlesignon/latest/userguide/getting-started.html) 配置用户和角色，并确保拥有可访问 Bedrock 的权限，例如 `PowerUserAccess`。
    - 如果你的 AWS 访问由公司提供，请在 AWS access portal 中选择目标账号，并确认权限满足 Bedrock 使用要求。
    - 打开 `Access keys` 链接，记录 `SSO start URL` 和 `SSO region`，下一步会用到。

3. 使用 [`aws configure sso` CLI wizard](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-sso.html#cli-configure-sso-configure) 继续配置 profile。

    - 配置完成后运行：`aws sso login --profile <AWS-profile-name>`。
    - 记录该 profile name，后续需要在 NBG 的 AWS Bedrock 配置中填写。

4. 如果尚未安装 NBG，请参考 [Getting Started](/getting-started/installing-cline)。

5. 打开 NBG 设置并选择 AWS Bedrock provider。
    - 在 "API Provider" 下拉菜单中选择 **AWS Bedrock**。
    - 选择 **AWS Profile** 认证方式，并填写第 3 步记录的 AWS profile name。
    - 在下拉菜单中选择 AWS Region。
    - 部分模型需要勾选 cross-region inference。
