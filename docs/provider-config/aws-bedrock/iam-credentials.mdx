### 概览

-   **AWS Bedrock：** AWS 托管的生成式 AI 服务，可访问 Anthropic Claude、Amazon Nova 等模型。\
    参考 [AWS Bedrock 文档](https://docs.aws.amazon.com/bedrock/latest/userguide/what-is-bedrock.html)。
-   **NBG：** 通过 provider 配置接入 Bedrock 模型，用于代码生成、调试、分析和自动化任务。
-   **适用场景：** 适合已有 AWS 环境的组织，例如使用 IAM roles、AWS SSO、AWS Organizations 的团队。

---

### Step 1: 准备 AWS 环境

#### 1.1 创建或使用 IAM role/user

1. **登录 AWS Management Console：**\
   [AWS Console](https://aws.amazon.com/console/)
2. **打开 IAM：**
    - 在 AWS Console 中搜索 **IAM (Identity and Access Management)**。
    - 创建新的 IAM user，或通过企业 AWS SSO assume 专用的 Bedrock access role。
    - [AWS IAM User Guide](https://docs.aws.amazon.com/IAM/latest/UserGuide/introduction.html)

#### 1.2 绑定所需 policy

为了让 NBG 调用 AWS Bedrock，IAM user 或 role 需要相应权限。`AmazonBedrockLimitedAccess` managed policy 覆盖范围较广；如果希望遵循最小权限原则，核心模型调用通常需要：

-   `bedrock:InvokeModel`
-   `bedrock:InvokeModelWithResponseStream`

你可以创建自定义 IAM policy，并将其绑定到对应 IAM user 或 role。

**选项 1：最小权限（推荐用于生产和最小权限场景）**

1.  在 AWS IAM console 中创建新 policy。
2.  使用 JSON editor 添加以下 policy document：
    ```json
    {
        "Version": "2012-10-17",
        "Statement": [
            {
                "Effect": "Allow",
                "Action": ["bedrock:InvokeModel", "bedrock:InvokeModelWithResponseStream"],
                "Resource": "*"
            }
        ]
    }
    ```
3.  为 policy 命名，例如 `NBGBedrockInvokeAccess`，并绑定到目标 IAM user 或 role。

**选项 2：使用 managed policy（初始配置更简单）**

-   也可以绑定 AWS managed policy **`AmazonBedrockLimitedAccess`**。它授予更广的权限，包括列出模型、管理 provisioning 以及其他 Bedrock 功能。初始配置或需要更完整能力时更简单。
    [View AmazonBedrockLimitedAccess Policy Details](https://docs.aws.amazon.com/bedrock/latest/userguide/security-iam.html)

**重要说明：**

-   **模型列表：** 如果在 NBG 中直接填写 model ID，最小权限通常足够调用模型。若需要动态列出可用 Bedrock 模型，可能还需要 `bedrock:ListFoundationModels`。
-   **AWS Marketplace 订阅：** 第三方模型，例如 Anthropic Claude，通常需要在 AWS Bedrock console 的 "Model access" 中完成订阅；必要时还需要 `aws-marketplace:Subscribe`。
-   **企业建议：** 始终使用最小权限。可行时将 IAM policy 的 resource ARNs 限制到具体模型或区域，并结合 [Service Control Policies (SCPs)](https://docs.aws.amazon.com/organizations/latest/userguide/orgs_manage_policies_scps.html) 做组织级治理。

### Step 2: 确认区域访问

#### 2.1 选择并确认 Region

1. **选择 Region：**\
   AWS Bedrock 在多个区域可用，例如 US East、Europe、Asia Pacific。根据延迟、合规和团队策略选择区域。\
   [AWS Global Infrastructure](https://aws.amazon.com/about-aws/global-infrastructure/regions_az/)
2. **验证模型访问：**
    - 在 AWS Bedrock console 中确认团队所需模型，例如 Anthropic Claude、Amazon Nova，状态为 "Access granted"。
    - 部分高级模型如果不支持 on-demand 访问，可能需要 [Inference Profile](https://docs.aws.amazon.com/bedrock/latest/userguide/inference-profiles-support.html)。

#### 2.2 配置 AWS Marketplace 订阅（如需要）

1. **订阅第三方模型：**
    - 打开 AWS Bedrock console，找到模型订阅区域。
    - 对于第三方 provider 的模型，例如 Anthropic，接受条款并完成订阅。
    - [AWS Marketplace](https://aws.amazon.com/marketplace/)
2. **企业建议：**
    - 模型订阅通常由云平台团队集中管理。请确认内部是否已有标准订阅流程。

### Step 3: 在 NBG 中配置

#### 3.1 安装并打开 NBG

如果尚未安装 NBG，请参考 [Getting Started](/getting-started/installing-cline)。

#### 3.2 配置 NBG 设置

1. **打开 NBG 设置：**
    - 点击设置图标 ⚙️ 进入 API Provider 配置。
2. **选择 AWS Bedrock provider：**
    - 在 "API Provider" 下拉菜单中选择 **AWS Bedrock**。
3. **填写 AWS credentials：**
    - 输入 **Access Key** 和 **Secret Key**。如果使用 AWS SSO，可按团队规范使用临时凭证。
    - 选择正确的 **AWS Region**，例如 `us-east-1` 或企业批准的区域。
4. **选择模型：**
    - 选择可用的 Bedrock provider model。
5. **保存并测试：**
    - 点击 **Done/Save** 应用设置。
    - 发送一个简单请求验证连接，例如生成一个判断质数的 Python 函数。

---

### Step 4: 安全、监控与最佳实践

1. **访问安全：**
    - 优先使用 AWS SSO/federated roles，而不是长期 IAM credentials。
    - [AWS IAM Best Practices](https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html)
2. **网络安全：**
    - 可考虑使用 [AWS PrivateLink](https://docs.aws.amazon.com/vpc/latest/userguide/endpoint-services-overview.html) 安全连接 Bedrock。
3. **监控与日志：**
    - 启用 AWS CloudTrail 记录 Bedrock API 调用。
    - 使用 CloudWatch 监控调用次数、延迟和 token 用量。
    - 为异常活动设置告警。
4. **错误处理与成本管理：**
    - 对 throttling errors 实现指数退避。
    - 使用 AWS Cost Explorer 和账单告警跟踪使用量。\
      [AWS Cost Management](https://docs.aws.amazon.com/cost-management/latest/userguide/what-is-aws-cost-management.html)
5. **审计与合规：**
    - 定期审查 IAM roles 和 CloudTrail logs。
    - 遵守内部数据隐私和治理策略。
