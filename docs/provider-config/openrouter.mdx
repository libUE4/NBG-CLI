---
title: "OpenRouter"
description: "在 NBG 中通过 OpenRouter 使用多个模型 provider。"
---

OpenRouter 通过统一 API 提供多个 provider 的语言模型访问能力，便于在 NBG 中快速切换和评估不同模型。

**Website:** [https://openrouter.ai/](https://openrouter.ai/)

### 获取 API 凭证

1.  **注册或登录：** 打开 [OpenRouter website](https://openrouter.ai/)，使用 Google 或 GitHub 账号登录。
2.  **获取 API key：** 打开 [keys page](https://openrouter.ai/keys)。如果没有现有密钥，请创建一个新密钥。
3.  **复制密钥：** 复制 API key，并妥善保存。

### 在 NBG 中配置

> 参考 [认证与模型选择](/getting-started/authorizing-with-cline#menu)。

1.  **打开 NBG 设置：** 在 NBG 面板中点击设置图标 (⚙️)。
2.  **选择 provider：** 在 "API Provider" 下拉菜单中选择 "OpenRouter"。
3.  **填写 API key：** 将 OpenRouter API key 粘贴到 "OpenRouter API Key" 字段。
4.  **选择模型：** 在 "Model" 下拉菜单中选择需要使用的模型。
5.  **可选，自定义 Base URL：** 如需使用 OpenRouter API 的自定义 base URL，勾选 "Use custom base URL" 并填写地址。大多数用户无需修改。

### 提示

-   **价格：** OpenRouter 按底层模型价格计费，详见 [OpenRouter Models page](https://openrouter.ai/models)。
-   **缓存：** OpenRouter 会将缓存请求传递给支持缓存的底层模型。可在 [OpenRouter Models page](https://openrouter.ai/models) 查看哪些模型支持缓存。
-   **Gemini 特例：** 通过 OpenRouter 使用 Gemini 模型时，Google 缓存机制偶尔可能导致响应延迟，因此需要在 provider 设置中手动勾选 "Enable Prompt Caching"。非 Gemini 模型通常无需勾选该选项。
