<Warning>
  插件功能当前适用于 NBG CLI、SDK 和 Kanban，暂不适用于 VSCode 与 JetBrains 扩展。
</Warning>

插件可以为 NBG 增加自定义工具、生命周期 hooks、斜杠命令等能力。插件既可以全局安装，也可以按项目安装。

## 通过 CLI 安装插件

`nbg plugin install` 支持四类来源：

<Tabs>
  <Tab title="File URL">
    ```bash
    nbg plugin install https://github.com/owner/repo/blob/main/plugins/my-plugin.ts
    nbg plugin install https://raw.githubusercontent.com/owner/repo/main/plugins/my-plugin.ts
    ```

    文件 URL 会直接安装单个 `.ts` 或 `.js` 插件文件。支持 GitHub `blob` 和 raw URL，远程插件文件必须使用 `https://`。
  </Tab>
  <Tab title="Git Repository">
    ```bash
    nbg plugin install https://github.com/owner/repo.git
    nbg plugin install git@github.com:owner/repo.git
    ```

    安装器会克隆仓库、安装生产依赖，并注册插件入口文件。

    如需安装指定分支或 tag，在末尾追加 `@ref`：

    ```bash
    nbg plugin install https://github.com/owner/repo.git@v1.2.0
    nbg plugin install https://github.com/owner/repo.git@main
    ```
  </Tab>
  <Tab title="npm Package">
    ```bash
    nbg plugin install npm:@scope/my-plugin
    nbg plugin install --npm my-plugin
    ```
  </Tab>
  <Tab title="Local Path">
    ```bash
    nbg plugin install ./my-plugin
    nbg plugin install ~/plugins/my-tool
    nbg plugin install /absolute/path/to/plugin.ts
    ```

    本地安装会把文件或目录复制到插件存储区。支持单个 `.ts`/`.js` 文件，也支持带 `package.json` 的目录。
  </Tab>
</Tabs>

常用参数：

| 参数 | 说明 |
|------|-------------|
| `--force` | 替换同一 source 的已有安装 |
| `--json` | 以 JSON 输出结果，便于脚本调用 |
| `--cwd <path>` | 安装到 `<path>/.cline/plugins`，而不是全局目录 |

安装后运行 `nbg config`，在插件信息中确认插件已加载。

### 示例：TypeScript 导航插件

仓库内的 [`typescript-lsp`](https://github.com/libUE4/NBG-CLI/tree/main/sdk/examples/plugins/typescript-lsp) 示例展示了插件如何工作。它会增加 `goto_definition` 工具，通过 TypeScript Language Service API 解析 import、re-export 和 type alias 后的符号定义。

在仓库根目录运行：

```bash
nbg plugin install ./sdk/examples/plugins/typescript-lsp
```

安装后，NBG 可以使用文件路径和行号调用 `goto_definition`，比纯文本搜索更精确地定位符号定义。

## 插件 Manifest 格式

为保持兼容，插件仓库或 npm 包仍通过 `package.json` 中的 `cline` 字段声明入口：

```json
{
  "name": "my-nbg-plugin",
  "version": "1.0.0",
  "cline": {
    "plugins": [
      {
        "paths": ["./index.ts"],
        "capabilities": ["tools", "hooks"]
      }
    ]
  }
}
```

`cline.plugins` 数组支持：

| 格式 | 示例 |
|------|------|
| 带 `paths` 数组的对象 | `{ "paths": ["./src/plugin.ts"], "capabilities": ["tools"] }` |
| 字符串路径 | `"./index.ts"` |

每个路径应指向导出 `AgentPlugin` 的 `.ts` 或 `.js` 文件，可以是 default export，也可以是 named export。

如果没有 `cline.plugins` 字段，安装器会回退到自动发现：先查找标准入口，再递归扫描 `.ts` 和 `.js` 文件（跳过 `node_modules` 与 `.git`）。

### Host 提供的依赖

`@cline/` scope 下的依赖由当前兼容运行时提供。安装器在执行 `npm install` 前会从插件依赖列表中移除这些包，因此插件导入的 `@cline/*` 包应声明为可选 peer dependency。当前 host 提供 `@cline/sdk`、`@cline/core`、`@cline/agents`、`@cline/llms` 和 `@cline/shared`。

```json
{
  "peerDependencies": {
    "@cline/sdk": "*"
  },
  "peerDependenciesMeta": {
    "@cline/sdk": {
      "optional": true
    }
  }
}
```

## 插件目录结构

插件存储在两级 `plugins` 目录中：

```
~/.cline/
  plugins/                     # 全局插件
    _installed/                # 由 `nbg plugin install` 管理
      npm/                     # npm 来源插件
      git/                     # git 来源插件
      remote/                  # 文件 URL 来源插件
      local/                   # 本地来源插件

.cline/                        # 项目根目录
  plugins/                     # 项目级插件
```

全局插件（`~/.cline/plugins/`）对所有会话可用。项目插件（仓库内 `.cline/plugins/`）只在该项目中可用。
