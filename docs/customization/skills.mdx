Skills 是用于特定任务的模块化指令集，用来扩展 NBG 能力。每个 skill 都可以打包详细指导、流程和可选资源，NBG 只会在与你的请求相关时加载它们。

可以安装多个 skills，NBG 只加载当前需要的内容。部署 skill 会保持休眠，直到你提出部署相关请求。不同于始终生效的 [rules](/customization/cline-rules)，skills 按需加载，因此处理无关任务时不会占用上下文。

<Note>
Skills 是实验性功能。可在 **Settings → Features → Enable Skills** 中启用。
</Note>

## Skills 如何工作

Skills 使用 progressive loading 来提高效率：

| 层级 | 加载时机 | Token 成本 | 内容 |
|-------|-------------|------------|---------|
| Metadata | 始终加载（启动时） | 每个 skill 约 100 tokens | YAML frontmatter 中的 `name` 和 `description` |
| Instructions | Skill 被触发时 | 低于 5k tokens | `SKILL.md` 正文中的说明和指导 |
| Resources | 按需加载 | 实际上不受限制 | 通过 `read_file` 访问的 bundled files 或可执行 scripts |

发送消息时，NBG 会看到可用 skills 及其 descriptions。如果请求匹配某个 skill 的 description，NBG 会使用 `use_skill` tool 激活它，并从 `SKILL.md` 加载完整 instructions。

## 用 Slash Commands 触发 Skills

也可以在聊天输入框中使用 slash commands 显式调用已启用的 skills。

1. 在聊天中输入 `/` 打开 command suggestions。
2. 选择要运行的 skill command，例如 `/aws-deploy`。
3. NBG 会触发该 skill，并加载它的 `SKILL.md` instructions。

当你希望立即强制使用某个 skill，而不是等待基于 description 的自动匹配时，这很有用。

## Skill 结构

每个 skill 都是一个目录，其中包含带 YAML frontmatter 的 `SKILL.md` 文件。

```text title="Skill directory structure"
my-skill/
├── SKILL.md          # Required: main instructions
├── docs/             # Optional: additional documentation
│   └── advanced.md
└── scripts/          # Optional: utility scripts
    └── helper.sh
```

`SKILL.md` 文件包含两部分：metadata 和 instructions。

```markdown title="SKILL.md"
---
name: my-skill
description: Brief description of what this skill does and when to use it.
---

# My Skill

Detailed instructions for NBG to follow when this skill is activated.

## Steps
1. First, do this
2. Then do that
3. For advanced usage, see [advanced.md](docs/advanced.md)
```

必填字段：

- `name` 必须与目录名完全一致
- `description` 告诉 NBG 何时使用该 skill，最多 1024 个字符

## 创建 Skill

<Steps>
  <Step title="打开 Skills menu">
    点击 NBG panel 底部、model selector 左侧的 scale icon。切换到 Skills tab。
  </Step>
  <Step title="创建新 skill">
    点击 **New skill...** 并输入 skill 名称，例如 `aws-deploy`。NBG 会创建 skill directory，并生成 template `SKILL.md` 文件。
  </Step>
  <Step title="编写 skill instructions">
    编辑 `SKILL.md` 文件：
    - 更新 `description` 字段，说明该 skill 何时应该触发
    - 在正文中添加详细 instructions
    - 可选地在 `docs/`、`templates/` 或 `scripts/` 子目录中添加 supporting files
  </Step>
</Steps>

也可以在文件系统中手动创建目录结构来创建 skills。将 skill directories 放到兼容的 workspace 路径 `.cline/skills/` 或 global 路径 `~/.cline/skills/` 中，NBG 会自动检测。

把重要信息放在 `SKILL.md` 前面。NBG 会按顺序读取文件，因此常见场景应尽量前置。使用清晰的 section headers，例如 `## Error Handling` 或 `## Configuration`，便于 NBG 快速定位相关部分。

### 启用或禁用 Skills

每个 skill 都有一个 toggle，用于启用或禁用。这样可以在不删除 skill directory 的情况下控制哪些 skills 生效。Skills 被发现后默认启用。

例如，本地开发时可以禁用 CI/CD skill；只在处理特定客户项目时启用 client-specific skill。

## 编写 SKILL.md

### 命名约定

Skill 名称出现在 `name` 字段中，必须与目录名完全一致。使用带连字符的小写形式（kebab-case），并清楚描述该 skill 的用途。

好的名称：
- `aws-cdk-deploy`
- `pr-review-checklist`
- `database-migration`
- `api-client-generator`

避免：
- `aws`（太模糊）
- `my_skill`（使用下划线，且描述性不足）
- `DeployToAWS`（应使用 kebab-case，而不是 PascalCase）
- `misc-helpers`（太泛化）

### 编写有效 Descriptions

`description` 决定 NBG 何时激活 skill。描述模糊时，skill 可能不会在你期望的时机触发。

好的 descriptions 应该具体且可执行：

```yaml
description: Deploy applications to AWS using CDK. Use when deploying, updating infrastructure, or managing AWS resources.

description: Generate release notes from git commits. Use when preparing releases, writing changelogs, or summarizing recent changes.

description: Analyze CSV and Excel data files. Use when exploring datasets, generating statistics, or creating visualizations from tabular data.
```

弱 descriptions 会留下太多歧义：

```yaml
description: Helps with AWS stuff.

description: Data analysis helper.

description: Useful for releases.
```

先说明 skill 做什么，尽量使用动作动词；加入用户可能使用的 trigger phrases，并提到具体 file types、tools 或 domains。可以用不同请求措辞测试 descriptions，确认 skill 会按预期触发。

### 保持 Skills 聚焦

尽量让 `SKILL.md` 低于 5k tokens。如果 skill 需要更多内容，把它们拆到 `docs/` 目录中的独立文件，并从主 instructions 引用。NBG 只会在需要时加载被引用文件。

加入真实示例。展示要运行的命令、预期输出和结果形态。相比抽象说明，具体示例更容易执行。

## Skills 存放位置

Skills 可以全局存放，也可以放在 project workspace 中。不同位置的使用建议见 [Storage Locations](/getting-started/config#storage-locations)。

Project skills：
- `.cline/skills/` (recommended)
- `.clinerules/skills/`
- `.claude/skills/`

Global skills：
- `~/.cline/skills/` (macOS/Linux)
- `C:\Users\USERNAME\.cline\skills\` (Windows)

当 global skill 和 project skill 同名时，global skill 优先。这让你可以把通用 skills 放在全局，同时把 project-specific skills 放在兼容的 `.cline/skills/` 路径中，供整个团队使用。

通过提交 `.cline/skills/` 来对 project skills 做版本控制。团队可以共同共享、review 和改进这些 skills。

## 打包 Supporting Files

Skills 可以包含额外文件，NBG 只在需要时访问。

```text title="Directory structure"
complex-skill/
├── SKILL.md
├── docs/
│   ├── setup.md
│   └── troubleshooting.md
├── templates/
│   └── config.yaml
└── scripts/
    └── validate.py
```

### docs/

使用 `docs/` 存放过于详细、不适合放进 `SKILL.md`，或只在特定场景相关的信息：

- Advanced configuration options
- Edge cases 的 troubleshooting guides
- Reference material，例如 API schemas、database schemas
- Platform-specific instructions

部署 skill 可以包含 `docs/aws.md`、`docs/gcp.md` 和 `docs/azure.md`。NBG 会根据你的请求只加载相关平台指南。

### templates/

当 skill 需要创建 configuration files、boilerplate code 或 structured documents 时，使用 `templates/`：

- Config files，例如 Terraform、Docker Compose、CI/CD pipelines
- Code scaffolding，例如 component templates、test fixtures
- Documentation templates，例如 README、API docs

项目初始化 skill 可以包含 `templates/dockerfile`、`templates/docker-compose.yml` 和 `templates/.env.example`，由 NBG 按每个新项目定制。

### scripts/

对于需要稳定行为的 deterministic operations，使用 `scripts/`：

- Validation，例如 linting configs、checking prerequisites
- Data processing，例如 parsing、formatting、transforming
- Complex calculations，例如 cost estimation、resource sizing
- API interactions，例如 fetching data、running health checks

Scripts 很节省 tokens，因为只有输出会进入上下文，而不是脚本代码本身。一个 500 行 validation script 可以只产生简单的 `Passed` 或详细错误信息，而不会消耗脚本逻辑本身的上下文。

### 引用 Bundled Files

在 `SKILL.md` instructions 中引用这些文件：

```markdown title="SKILL.md (referencing bundled files)"
For initial setup, follow [setup.md](docs/setup.md).
Use the config template at `templates/config.yaml` as a starting point.
Run the validation script to check your configuration:

python scripts/validate.py
```

当 instructions 引用 documentation files 时，NBG 会使用 `read_file` 读取它们。Scripts 可以直接执行，且只有脚本输出进入 context window。

| 适合使用 Scripts | 适合使用 Instructions |
|-----------------|---------------------|
| Deterministic operations，例如 validation、formatting | 可根据上下文调整的灵活指导 |
| Complex computations | Decision-making processes |
| 需要可靠性的操作 | 会因情况变化的步骤 |
| 不希望用 tokens 解释的内容 | Best practices 和 patterns |

## 示例：Data Analysis Skill

下面是一个用于 data analysis tasks 的实用 skill。创建名为 `data-analysis/` 的目录，并加入这个 `SKILL.md`：

```markdown title="data-analysis/SKILL.md"
---
name: data-analysis
description: Analyze data files and generate insights. Use when working with CSV, Excel, or JSON data files that need exploration, cleaning, or visualization.
---

# Data Analysis

When analyzing data files, follow this process:

## 1. Understand the Data
- Read a sample of the file to understand its structure
- Identify column types and data quality issues
- Note any missing values or anomalies

## 2. Ask Clarifying Questions
Before diving in, ask the user:
- What specific insights are they looking for?
- Are there any known data quality issues?
- What format do they want for the output?

## 3. Perform Analysis
Use pandas for data manipulation:

import pandas as pd

# Load and explore
df = pd.read_csv("data.csv")
print(df.head())
print(df.describe())
print(df.info())

For visualization, prefer matplotlib or seaborn depending on complexity.
```

Skills 可以把 NBG 从 general-purpose assistant 变成了解你领域的 specialist。可以先为一个经常重复的任务创建 skill，测试它，并迭代 description，直到它能可靠触发。
